## 版本管理

本项目遵循一种稍作修改的语义化版本管理，采用 `0.Y.Z` 形式。前导的 `0` 表示 SDK 仍在快速演进中。各组件递增规则如下：

## 次版本号（`Y`）

对于未标记为 beta 的任何公共接口出现的**不兼容变更**，我们会提升次版本号 `Y`。例如，从 `0.0.x` 升级到 `0.1.x` 可能包含不兼容变更。

如果你不希望引入不兼容变更，建议在项目中固定到 `0.0.x` 版本。

## 修订号（`Z`）

对于不会引起不兼容的变更，我们会递增 `Z`：

- Bug 修复
- 新功能
- 私有接口变更
- beta 功能更新

## 子包的版本管理

主包 `@openai/agents` 由可独立使用的多个子包组成。目前各包的版本是联动的，即任一包版本提升时，其他包也会随之提升。随着我们迈向 `1.0.0`，该策略可能会调整。
