## 版本控制

该项目遵循经过轻微修改的语义化版本控制，形式为 `0.Y.Z`。前导的 `0` 表示 SDK 仍在快速演进中。各组件的递增方式如下：

## 次要（`Y`）版本

对于未标记为 beta 的任何公共接口出现的**不兼容变更**，我们会提升次要版本 `Y`。例如，从 `0.0.x` 到 `0.1.x` 可能包含不兼容变更。

如果你不希望引入不兼容变更，建议在项目中固定到 `0.0.x` 版本。

## 补丁（`Z`）版本

对于不引入不兼容的变更，我们会递增 `Z`：

- Bug 修复
- 新功能
- 私有接口的变更
- beta 功能的更新

## 子包版本控制

主包 `@openai/agents` 由多个可独立使用的子包组成。目前这些包的版本是联动的，即如果某个包提升了版本，其他包也会随之提升。随着我们迈向 `1.0.0`，这一策略可能会改变。
