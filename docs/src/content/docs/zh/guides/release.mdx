## 版本管理

该项目遵循经轻微修改的语义化版本控制，形式为 `0.Y.Z`。前导的 `0` 表示 SDK 仍在快速演进。各组件的递增规则如下：

## 次要版本（`Y`）

对于未标记为 beta 的任何公共接口的**破坏性变更**，我们将提升次要版本 `Y`。例如，从 `0.0.x` 到 `0.1.x` 可能包含破坏性变更。

如果你不希望引入破坏性变更，建议在项目中将版本固定到 `0.0.x`。

## 补丁版本（`Z`）

对于非破坏性变更，我们将递增 `Z`：

- Bug 修复
- 新功能
- 私有接口的变更
- 对 beta 功能的更新

## 子包版本管理

主包 `@openai/agents` 由多个可独立使用的子包组成。目前这些包的版本是联动的，这意味着如果一个包提升了版本，其他包也会随之提升。随着我们迈向 `1.0.0`，这一策略可能会改变。
