智能体是 OpenAI Agents SDK 的主要构建块。一个**Agent**是经过如下配置的大语言模型（LLM）：

- **Instructions**——用于告知模型“它是谁”以及“应如何回复”的系统提示。
- **Model**——要调用的 OpenAI 模型，以及可选的模型调优参数。
- **Tools**——LLM 可调用以完成任务的函数或 API 列表。

## 基础配置

`Agent` 构造函数接收一个配置对象。最常用的属性如下所示。

| Property        | Required | Description                                                                                |
| --------------- | -------- | ------------------------------------------------------------------------------------------ |
| `name`          | yes      | 简短、可读的标识符。                                                                       |
| `instructions`  | yes      | 系统提示（字符串**或**函数——参见[动态 instructions](#dynamic-instructions)）。             |
| `model`         | no       | 模型名称**或**自定义的 [`Model`](/openai-agents-js/openai/agents/interfaces/model/) 实现。 |
| `modelSettings` | no       | 调优参数（temperature、top_p 等）。如果所需属性不在顶层，可将其放在 `providerData` 下。    |
| `tools`         | no       | 模型可调用的 [`Tool`](/openai-agents-js/openai/agents/type-aliases/tool/) 实例数组。       |

## 上下文

## 输出类型

默认情况下，智能体返回**纯文本**（`string`）。如果希望模型返回结构化对象，可通过 `outputType` 属性指定。SDK 接受：

1. [Zod](https://github.com/colinhacks/zod) 模式（`z.object({...})`）。
2. 任何兼容 JSON Schema 的对象。

## 多智能体系统设计模式

组合智能体有多种方式。在生产应用中常见的两种模式是：

1. **管理者（智能体即工具）**——中心智能体掌控会话，并调用作为工具暴露的专业化智能体。
2. **交接**——初始智能体在识别出用户请求后，将整个会话交给专家智能体处理。

这两种方法是互补的。管理者让你可以在单一位置实施护栏或速率限制，而交接则使每个智能体专注于单一任务而无需持续掌控对话。

### 管理者（智能体即工具）

在该模式中，管理者不会移交控制权——LLM 使用工具，最终由管理者总结答案。更多内容见[工具](/openai-agents-js/zh/guides/tools#agents-as-tools)。

<Code lang="typescript" code={agentsAsTools} title="Agents as tools" />

### 交接

在交接模式下，分诊智能体负责路由请求，但一旦发生交接，专家智能体将接管对话直至产生最终输出。这样可以保持提示简短，并使你能够独立地推理每个智能体。了解更多见[交接](/openai-agents-js/zh/guides/handoffs)。

## 强制使用工具

提供工具并不保证 LLM 一定会调用。你可以通过 `modelSettings.tool_choice` 来**强制**使用工具：

1. `'auto'`（默认）——由 LLM 决定是否使用工具。
2. `'required'`——LLM 必须调用某个工具（可自行选择）。
3. `'none'`——LLM 必须**不**调用工具。
4. 指定工具名，如 `'calculator'`——LLM 必须调用该特定工具。

<Code lang="typescript" code={agentForcingToolUse} title="Forcing tool use" />

### 避免无限循环

在一次工具调用后，SDK 会自动将 `tool_choice` 重置为 `'auto'`。这可防止模型陷入反复尝试调用工具的无限循环。你可以通过 `resetToolChoice` 标志或配置 `toolUseBehavior` 覆盖该行为：

- `'run_llm_again'`（默认）——使用工具结果再次运行 LLM。
- `'stop_on_first_tool'`——将第一次工具结果视为最终答案。
- `{ stopAtToolNames: ['my_tool'] }`——当任一列出的工具被调用时停止。
- `(context, toolResults) => ...`——返回运行是否应结束的自定义函数。

```typescript
const agent = new Agent({
  ...,
  toolUseBehavior: 'stop_on_first_tool',
});
```
