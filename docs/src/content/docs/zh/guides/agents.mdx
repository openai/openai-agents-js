智能体是 OpenAI Agents SDK 的主要构建模块。**Agent** 是一个经过如下配置的大语言模型（LLM）：

- **Instructions**——系统提示，用于告诉模型*它是谁*以及*应如何响应*。
- **Model**——要调用的 OpenAI 模型，以及可选的模型调优参数。
- **Tools**——LLM 可调用的一组函数或 API，用于完成任务。

## 智能体基础

### 基础配置

`Agent` 构造函数接收一个配置对象。最常用的属性如下所示。

| Property                          | Required | Description                                                                                                         |
| --------------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------- |
| `name`                            | yes      | 简短、便于人类阅读的标识符。                                                                                        |
| `instructions`                    | yes      | 系统提示（字符串**或**函数——参见[动态说明](#dynamic-instructions)）。                                               |
| `prompt`                          | no       | OpenAI Responses API 提示配置。参见[Prompt](/openai-agents-js/zh/guides/models#prompt)。                            |
| `handoffDescription`              | no       | 当该智能体作为交接工具提供时使用的简短描述。                                                                        |
| `model`                           | no       | 模型名称**或**自定义 [`Model`](/openai-agents-js/openai/agents/interfaces/model/) 实现。                            |
| `modelSettings`                   | no       | 调优参数（temperature、top_p 等）。如果所需属性不在顶层，可放在 `providerData` 下。                                 |
| `tools`                           | no       | 模型可调用的 [`Tool`](/openai-agents-js/openai/agents/type-aliases/tool/) 实例数组。                                |
| `mcpServers`                      | no       | 为智能体提供工具的 MCP 服务器。参见 [MCP 集成](/openai-agents-js/zh/guides/mcp)。                                   |
| `resetToolChoice`                 | no       | 工具调用后将 `toolChoice` 重置为默认值（默认：`true`），以防止工具使用循环。参见[强制工具使用](#forcing-tool-use)。 |
| `handoffOutputTypeWarningEnabled` | no       | 当交接输出类型不同时发出警告（默认：`true`）。                                                                      |

### 上下文

智能体在其上下文类型上是**泛型**的——即 `Agent<TContext, TOutput>`。*上下文*是您创建并传给 `Runner.run()` 的依赖注入对象。它会传递给每个工具、护栏、交接等，适用于存储状态或提供共享服务（数据库连接、用户元数据、功能开关等）。

### 输出类型

默认情况下，智能体返回**纯文本**（`string`）。如果您希望模型返回结构化对象，可以指定 `outputType` 属性。SDK 支持：

1. [Zod](https://github.com/colinhacks/zod) schema（`z.object({...})`）。
2. 任意兼容 JSON schema 的对象。

## 多智能体系统中的智能体

当智能体参与更大工作流时，最常见的是以下两个 SDK 入口：

1. **管理者（agents as tools）**——由中心智能体掌管会话，并调用作为工具暴露的专业智能体。
2. **交接**——初始智能体在识别到用户请求后，将整个会话委托给专业智能体。

这两种方式是互补的。管理者让您能在单一位置实施护栏或速率限制，而交接让每个智能体专注单一任务而无需持续掌控会话。关于设计权衡和选择时机，参见[智能体编排](/openai-agents-js/zh/guides/multi-agent)。

#### 管理者（agents as tools）

在这种模式中，管理者不会移交控制权——LLM 使用工具，由管理者汇总最终答案。更多信息请参见[工具](/openai-agents-js/zh/guides/tools#agents-as-tools)。

<Code lang="typescript" code={agentsAsTools} title="Agents as tools" />

#### 交接

在交接模式中，分诊智能体负责路由请求，但一旦发生交接，专业智能体将掌管会话直到产出最终输出。这可以保持提示简短，并让您能独立分析每个智能体。更多信息请参见[交接](/openai-agents-js/zh/guides/handoffs)。

### 生命周期钩子

在高级用例中，您可以通过监听事件来观察 Agent 生命周期。

`Agent` 实例会为该特定实例发出生命周期事件；`Runner` 则会在整个运行范围内以单一事件流发出相同事件名。这在多智能体工作流中很有用，便于您在一个位置观察交接和工具调用。

共享事件名如下：

| Event              | Agent hook arguments                    | Runner hook arguments                          |
| ------------------ | --------------------------------------- | ---------------------------------------------- |
| `agent_start`      | `(context, agent, turnInput?)`          | `(context, agent, turnInput?)`                 |
| `agent_end`        | `(context, output)`                     | `(context, agent, output)`                     |
| `agent_handoff`    | `(context, nextAgent)`                  | `(context, fromAgent, toAgent)`                |
| `agent_tool_start` | `(context, tool, { toolCall })`         | `(context, agent, tool, { toolCall })`         |
| `agent_tool_end`   | `(context, tool, result, { toolCall })` | `(context, agent, tool, result, { toolCall })` |

### 强制工具使用

提供工具并不保证 LLM 会调用它。您可以通过 `modelSettings.toolChoice` **强制**工具使用：

1. `'auto'`（默认）——由 LLM 决定是否使用工具。
2. `'required'`——LLM *必须*调用某个工具（可自行选择具体工具）。
3. `'none'`——LLM **不得**调用工具。
4. 指定工具名，例如 `'calculator'`——LLM 必须调用该特定工具。

<Code lang="typescript" code={agentForcingToolUse} title="Forcing tool use" />

#### 防止无限循环

工具调用后，SDK 会自动将 `toolChoice` 重置为 `'auto'`。这可防止模型进入反复尝试调用工具的无限循环。您可以通过 `resetToolChoice` 标志，或配置 `toolUseBehavior` 来覆盖此行为：

- `'run_llm_again'`（默认）——携带工具结果再次运行 LLM。
- `'stop_on_first_tool'`——将首次工具结果视为最终答案。
- `{ stopAtToolNames: ['my_tool'] }`——当调用到列出的任一工具时停止。
- `(context, toolResults) => ...`——自定义函数，返回运行是否应结束。

```typescript
const agent = new Agent({
  ...,
  toolUseBehavior: 'stop_on_first_tool',
});
```

注意：`toolUseBehavior` 仅适用于**函数工具**。托管工具始终会返回模型进行处理。
