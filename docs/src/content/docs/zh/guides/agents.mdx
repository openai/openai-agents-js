智能体是 OpenAI Agents SDK 的主要构建块。一个**Agent** 是已经配置好的大型语言模型（LLM），包括：

- **Instructions**——告知模型“它是谁”以及“它应如何回复”的系统提示。
- **Model**——要调用的 OpenAI 模型，以及可选的模型调优参数。
- **Tools**——LLM 可调用以完成任务的函数或 API 列表。

## 基本配置

`Agent` 构造函数接收一个配置对象。最常用的属性如下。

| 属性            | 必填 | 描述                                                                                       |
| --------------- | ---- | ------------------------------------------------------------------------------------------ |
| `name`          | yes  | 短的人类可读标识符。                                                                       |
| `instructions`  | yes  | 系统提示（字符串**或**函数——参见[动态 instructions](#dynamic-instructions)）。             |
| `model`         | no   | 模型名称**或**自定义的 [`Model`](/openai-agents-js/openai/agents/interfaces/model/) 实现。 |
| `modelSettings` | no   | 调优参数（temperature、top_p 等）。如果所需属性不在顶层，可放在 `providerData` 下。        |
| `tools`         | no   | 模型可调用的 [`Tool`](/openai-agents-js/openai/agents/type-aliases/tool/) 实例数组。       |

## 上下文

## 输出类型

默认情况下，Agent 返回**纯文本**（`string`）。如果希望模型返回结构化对象，可以指定 `outputType` 属性。SDK 接受：

1. [Zod](https://github.com/colinhacks/zod) 模式（`z.object({...})`）。
2. 任何兼容 JSON Schema 的对象。

## 多智能体系统设计模式

有多种方式将智能体组合在一起。我们在生产应用中常见的两种模式是：

1. **管理器（智能体即工具）**——中心智能体拥有对话，并调用以工具形式暴露的专业智能体。
2. **交接**——初始智能体在识别出用户请求后，将整个对话交给某个专家智能体。

这两种方法是互补的。管理器为你提供一个统一入口来实施护栏或速率限制；而交接让每个智能体专注于单一任务，而无需持续控制对话。

### 管理器（智能体即工具）

在此模式中，管理器从不移交控制权——LLM 使用工具，管理器汇总最终答案。详见[工具](/openai-agents-js/zh/guides/tools#agents-as-tools)。

<Code lang="typescript" code={agentsAsTools} title="智能体作为工具" />

### 交接

使用交接时，分诊智能体负责路由请求，但一旦发生交接，专家智能体将拥有对话，直到产出最终结果为止。这样可以保持提示简短，并让你能够独立地推理每个智能体。了解更多见[交接](/openai-agents-js/zh/guides/handoffs)。

## 强制使用工具

提供工具并不保证 LLM 一定会调用。你可以通过 `modelSettings.tool_choice` **强制**使用工具：

1. `'auto'`（默认）——LLM 决定是否使用工具。
2. `'required'`——LLM 必须调用某个工具（可自行选择）。
3. `'none'`——LLM 必须不调用任何工具。
4. 指定工具名，例如 `'calculator'`——LLM 必须调用该特定工具。

<Code lang="typescript" code={agentForcingToolUse} title="强制使用工具" />

### 防止无限循环

在工具调用之后，SDK 会自动将 `tool_choice` 重置为 `'auto'`。这可以防止模型陷入反复尝试调用工具的无限循环。你可以通过 `resetToolChoice` 标志或配置 `toolUseBehavior` 来覆盖该行为：

- `'run_llm_again'`（默认）——使用工具结果再次运行 LLM。
- `'stop_on_first_tool'`——将第一个工具结果视为最终答案。
- `{ stopAtToolNames: ['my_tool'] }`——当调用到列表中的任一工具时停止。
- `(context, toolResults) => ...`——返回运行是否应结束的自定义函数。

```typescript
const agent = new Agent({
  ...,
  toolUseBehavior: 'stop_on_first_tool',
});
```
