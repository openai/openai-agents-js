智能体是 OpenAI Agents SDK 的主要构建块。一个**Agent** 是已经配置好的大型语言模型（LLM），包含：

- **Instructions** —— 告诉模型它是谁以及应该如何响应的系统提示（system prompt）。
- **Model** —— 要调用的 OpenAI 模型，以及可选的模型调参。
- **Tools** —— LLM 可调用以完成任务的函数或 API 列表。

## 基本配置

`Agent` 构造函数接收一个配置对象。下表是最常用的属性。

| Property                          | Required | Description                                                                                                            |
| --------------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------- |
| `name`                            | yes      | 简短、便于阅读的标识符。                                                                                               |
| `instructions`                    | yes      | 系统提示（字符串**或**函数——见[动态 instructions](#dynamic-instructions)）。                                           |
| `prompt`                          | no       | OpenAI Responses API 提示配置。参见[模型](/openai-agents-js/zh/guides/models#prompt)中的“提示”。                       |
| `handoffDescription`              | no       | 当该智能体作为交接工具提供时使用的简短描述。                                                                           |
| `model`                           | no       | 模型名称**或**自定义的[`Model`](/openai-agents-js/openai/agents/interfaces/model/)实现。                               |
| `modelSettings`                   | no       | 调参（temperature、top_p 等）。如果你需要的属性不在顶层，可以放在 `providerData` 下。                                  |
| `tools`                           | no       | 模型可调用的[`Tool`](/openai-agents-js/openai/agents/type-aliases/tool/)实例数组。                                     |
| `mcpServers`                      | no       | 向智能体提供工具的 MCP 服务器。参见 [MCP 集成](/openai-agents-js/zh/guides/mcp)。                                      |
| `resetToolChoice`                 | no       | 在工具调用后将 `tool_choice` 重置为默认值（默认：`true`），以防止工具使用循环。参见[强制使用工具](#forcing-tool-use)。 |
| `handoffOutputTypeWarningEnabled` | no       | 当交接的输出类型不一致时发出警告（默认：`true`）。                                                                     |

## 上下文

智能体对其上下文类型是**泛型**的 —— 即 `Agent<TContext, TOutput>`。上下文（context）
是你创建并传给 `Runner.run()` 的依赖注入对象。它会被转发到每个工具、护栏、交接等，
可用于存储状态或提供共享服务（数据库连接、用户元数据、功能开关等）。

## 输出类型

默认情况下，智能体返回**纯文本**（`string`）。如果你希望模型返回结构化对象，可以指定
`outputType` 属性。SDK 接受：

1. [Zod](https://github.com/colinhacks/zod) schema（`z.object({...})`）。
2. 任何兼容 JSON Schema 的对象。

## 多智能体系统设计模式

有很多方式可以组合智能体。在生产应用中我们常见两种模式：

1. **Manager (agents as tools)** —— 一个中心智能体拥有对话，并调用作为工具暴露的专业智能体。
2. **交接（Handoffs）** —— 初始智能体在识别出用户请求后，将整个对话委托给专家智能体。

这两种方法是互补的。Manager 为你提供一个统一的位置来实施护栏或速率限制，而交接让每个智能体专注于单一任务，而不需要持续控制对话。

### 管理器（agents as tools）

在这种模式下，管理器不会移交控制权——LLM 使用工具，而管理器汇总最终答案。详见[工具](/openai-agents-js/zh/guides/tools#agents-as-tools)。

<Code lang="typescript" code={agentsAsTools} title="Agents as tools" />

### 交接

使用交接时，分诊智能体负责路由请求，但一旦发生交接，专家智能体就拥有对话控制权，直到产出最终输出为止。这样可以保持提示简短，并让你能够独立推理每个智能体。了解更多见[交接](/openai-agents-js/zh/guides/handoffs)。

## 强制使用工具

提供工具并不保证 LLM 会调用。你可以通过 `modelSettings.tool_choice` **强制**使用工具：

1. `'auto'`（默认）—— LLM 决定是否使用工具。
2. `'required'` —— LLM 必须调用某个工具（它可以自行选择）。
3. `'none'` —— LLM 必须**不**调用工具。
4. 指定工具名，例如 `'calculator'` —— LLM 必须调用该特定工具。

<Code lang="typescript" code={agentForcingToolUse} title="强制使用工具" />

### 防止无限循环

在一次工具调用后，SDK 会自动将 `tool_choice` 重置为 `'auto'`。这可防止模型反复尝试调用同一工具而陷入无限循环。你可以通过 `resetToolChoice` 标志覆盖这一行为，或通过配置
`toolUseBehavior` 来控制：

- `'run_llm_again'`（默认）—— 使用工具结果再次运行 LLM。
- `'stop_on_first_tool'` —— 将第一个工具结果作为最终答案。
- `{ stopAtToolNames: ['my_tool'] }` —— 当调用到列表中的任一工具时停止。
- `(context, toolResults) => ...` —— 返回是否应结束运行的自定义函数。

```typescript
const agent = new Agent({
  ...,
  toolUseBehavior: 'stop_on_first_tool',
});
```

注意：`toolUseBehavior` 仅适用于**函数工具（function tools）**。托管工具（Hosted tool）始终会返回给模型进行处理。
