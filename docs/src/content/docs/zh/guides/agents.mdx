智能体是 OpenAI Agents SDK 的主要构建模块。一个**智能体**是经过如下配置的大语言模型（LLM）：

- **Instructions** —— 指示模型“它是谁”以及“应该如何回应”的系统提示。
- **Model** —— 要调用的 OpenAI 模型，以及可选的模型调参。
- **Tools** —— LLM 为完成任务可以调用的一组函数或 API。

## 基本配置

`Agent` 构造函数只接受一个配置对象。下方展示最常用的属性。

| Property        | Required | Description                                                                                |
| --------------- | -------- | ------------------------------------------------------------------------------------------ |
| `name`          | yes      | 简短、可读的人类标识符。                                                                   |
| `instructions`  | yes      | 系统提示（字符串**或**函数——参见[动态 instructions](#dynamic-instructions)）。             |
| `model`         | no       | 模型名称**或**自定义的 [`Model`](/openai-agents-js/openai/agents/interfaces/model/) 实现。 |
| `modelSettings` | no       | 调参（temperature、top_p 等）。如果你需要的属性不在顶层，可以放在 `providerData` 下。      |
| `tools`         | no       | 模型可调用的 [`Tool`](/openai-agents-js/openai/agents/type-aliases/tool/) 实例数组。       |

## 上下文

智能体对其上下文类型是**泛型**的——即 `Agent<TContext, TOutput>`。上下文是你创建并传入
`Runner.run()` 的一个依赖注入对象。它会被转发到每个工具、护栏、交接等组件中，适合用于存储状态或提供共享服务（数据库连接、用户元数据、功能开关等）。

## 输出类型

默认情况下，智能体返回**纯文本**（`string`）。如果你希望模型返回结构化对象，可以指定 `outputType` 属性。SDK 接受：

1. [Zod](https://github.com/colinhacks/zod) 模式（`z.object({...})`）。
2. 任意兼容 JSON Schema 的对象。

## 多智能体系统设计模式

组合智能体的方式有很多。我们在生产应用中常见的两种模式是：

1. **管理者（智能体即工具）**——一个中心智能体拥有会话，并调用作为工具暴露的专业智能体。
2. **交接**——初始智能体在识别出用户请求后，将整个会话委托给某个专家智能体。

这些方法是互补的。管理者为你提供一个统一位置来实施护栏或速率限制，而交接让每个智能体专注于单一任务，而无需保留对会话的控制权。

### 管理者（智能体即工具）

在这种模式下，管理者不会移交控制权——LLM 使用这些工具，而管理者汇总最终答案。详见[工具](/openai-agents-js/zh/guides/tools#agents-as-tools)。

<Code lang="typescript" code={agentsAsTools} title="智能体即工具" />

### 交接

在交接模式中，分诊智能体负责路由请求，但一旦发生交接，专家智能体就拥有会话控制权，直到其产生最终输出。这可保持提示简短，并使你能独立地推理每个智能体。了解更多请参见[交接](/openai-agents-js/zh/guides/handoffs)。

## 强制使用工具

提供工具并不保证 LLM 一定会调用。你可以通过 `modelSettings.tool_choice` 来**强制**使用工具：

1. `'auto'`（默认）——由 LLM 决定是否使用工具。
2. `'required'`——LLM 必须调用某个工具（可自行选择）。
3. `'none'`——LLM 必须**不**调用工具。
4. 指定工具名，例如 `'calculator'`——LLM 必须调用该特定工具。

<Code lang="typescript" code={agentForcingToolUse} title="强制使用工具" />

### 防止无限循环

在一次工具调用后，SDK 会自动将 `tool_choice` 重置为 `'auto'`。这可防止模型陷入反复尝试调用工具的无限循环。你可以通过 `resetToolChoice` 标志或配置 `toolUseBehavior` 来覆盖此行为：

- `'run_llm_again'`（默认）——使用工具结果再次运行 LLM。
- `'stop_on_first_tool'`——将第一次工具结果视为最终答案。
- `{ stopAtToolNames: ['my_tool'] }`——当调用任一列出的工具时停止。
- `(context, toolResults) => ...`——返回是否应结束运行的自定义函数。

```typescript
const agent = new Agent({
  ...,
  toolUseBehavior: 'stop_on_first_tool',
});
```
