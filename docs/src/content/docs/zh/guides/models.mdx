## 模型选择

### 默认模型

当您在初始化 `Agent` 时未指定模型，将使用默认模型。当前默认是 [`gpt-4.1`](https://developers.openai.com/api/docs/models/gpt-4.1)，以兼顾兼容性与低延迟。如果您有权限，我们建议将智能体设置为 [`gpt-5.4`](https://developers.openai.com/api/docs/models/gpt-5.4)，以获得更高质量，同时显式设置 `modelSettings`。

如果您想切换到 [`gpt-5.4`](https://developers.openai.com/api/docs/models/gpt-5.4) 等其他模型，可以通过两种方式配置智能体。

第一种，如果您希望所有未设置自定义模型的智能体统一使用某个特定模型，请在运行智能体前设置 `OPENAI_DEFAULT_MODEL` 环境变量。

```bash
export OPENAI_DEFAULT_MODEL=gpt-5.4
node my-awesome-agent.js
```

第二种，您可以为 `Runner` 实例设置默认模型。如果智能体未设置模型，则会使用该 `Runner` 的默认模型。

如果您传入的是非 GPT-5 的模型名称，且未提供自定义 `modelSettings`，SDK 会回退到与任意模型兼容的通用 `modelSettings`。

## OpenAI provider 配置

### OpenAI provider

默认的 `ModelProvider` 使用 OpenAI API 解析模型名称。它支持两个不同端点：

| API | 用法 | 调用 `setOpenAIAPI()` |
| --- | --- | --- |
| Chat Completions | 标准对话与函数调用 | `setOpenAIAPI('chat_completions')` |
| Responses | 新的流式优先生成式 API（工具调用、灵活输出） | `setOpenAIAPI('responses')` _(默认)_ |

#### Responses WebSocket 传输

当您将 OpenAI provider 与 Responses API 一起使用时，可通过 WebSocket 传输发送请求，而不是默认的 HTTP 传输。

可通过 `setOpenAIResponsesTransport('websocket')` 全局启用，也可在 provider 级别通过 `new OpenAIProvider({ useResponses: true, useResponsesWebSocket: true })` 启用。

如果仅需使用 WebSocket 传输，您不需要 `withResponsesWebSocketSession(...)` 或自定义 `OpenAIProvider`。若每次运行/请求重新连接可接受，则启用 `setOpenAIResponsesTransport('websocket')` 后，现有 `run()` / `Runner.run()` 用法仍可继续工作。

传输方式选择遵循模型解析逻辑：

- `setOpenAIResponsesTransport('websocket')` 只影响后续通过 OpenAI provider 解析、且使用 Responses API 的字符串模型名称。
- 如果您向 `Agent` 或 `Runner` 传入具体的 `Model` 实例，则会原样使用该实例。`OpenAIResponsesWSModel` 继续走 WebSocket，`OpenAIResponsesModel` 继续走 HTTP，`OpenAIChatCompletionsModel` 继续走 Chat Completions。
- 如果您提供了自己的 `modelProvider`，则模型解析由该 provider 控制。请在其中启用 WebSocket，而不是依赖全局设置器。
- 如果您通过代理、网关或其他 OpenAI 兼容端点路由，请确保目标支持 WebSocket `/responses` 端点。您还可能需要显式设置 `websocketBaseURL`。

仅当您希望优化连接复用并更显式管理 websocket provider 生命周期时，才使用 `withResponsesWebSocketSession(...)` 或自定义 `OpenAIProvider` / `Runner`：

- `withResponsesWebSocketSession(...)`：便捷的作用域生命周期，回调结束后自动清理。
- 自定义 `OpenAIProvider` / `Runner`：在您自己的应用架构中进行显式生命周期控制（包括关闭清理）。

尽管名称中包含 Session，`withResponsesWebSocketSession(...)` 是传输生命周期辅助工具，与[会话指南](/openai-agents-js/zh/guides/sessions)中描述的内存 `Session` 接口无关。

如果您使用 websocket 代理或网关，请在 `OpenAIProvider` 上配置 `websocketBaseURL`，或设置 `OPENAI_WEBSOCKET_BASE_URL`。

如果您自行实例化 `OpenAIProvider`，请注意：基于 websocket 的 Responses 模型包装器默认会缓存以复用连接。请在关闭时调用 `await provider.close()` 释放这些缓存连接。`withResponsesWebSocketSession(...)` 很大程度上就是为您管理该生命周期：它会创建启用 websocket 的 provider 和 runner，将其传入回调，并在回调后始终关闭 provider。对临时 provider 使用 `providerOptions`，对回调作用域 runner 默认值使用 `runnerConfig`。

完整的流式传输 + HITL 示例（使用 Responses WebSocket 传输）请见 [`examples/basic/stream-ws.ts`](https://github.com/openai/openai-agents-js/tree/main/examples/basic/stream-ws.ts)。

## 模型行为与提示词

### ModelSettings

`ModelSettings` 映射 OpenAI 参数，但与 provider 无关。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `temperature` | `number` | 创造性与确定性之间的平衡。 |
| `topP` | `number` | Nucleus 采样。 |
| `frequencyPenalty` | `number` | 惩罚重复 token。 |
| `presencePenalty` | `number` | 鼓励生成新 token。 |
| `toolChoice` | `'auto' \| 'required' \| 'none' \| string` | 参见[强制工具使用](/openai-agents-js/zh/guides/agents#forcing-tool-use)。 |
| `parallelToolCalls` | `boolean` | 在支持时允许并行函数调用。 |
| `truncation` | `'auto' \| 'disabled'` | token 截断策略。 |
| `maxTokens` | `number` | 响应中的最大 token 数。 |
| `store` | `boolean` | 持久化响应，用于检索 / RAG 工作流。 |
| `promptCacheRetention` | `'in-memory' \| '24h' \| null` | 在支持时控制 provider 的提示缓存保留策略。 |
| `reasoning.effort` | `'none' \| 'minimal' \| 'low' \| 'medium' \| 'high' \| 'xhigh'` | gpt-5.x 模型的推理强度。 |
| `reasoning.summary` | `'auto' \| 'concise' \| 'detailed'` | 控制模型返回多少推理摘要。 |
| `text.verbosity` | `'low' \| 'medium' \| 'high'` | gpt-5.x 等模型的文本详略程度。 |
| `providerData` | `Record<string, any>` | 透传到底层模型的 provider 特定选项。 |

可在任一层级附加设置：

### Prompt

智能体可通过 `prompt` 参数进行配置，表示应使用服务器端存储的提示配置来控制智能体行为。目前，该选项仅在您使用 OpenAI [Responses API](https://platform.openai.com/docs/api-reference/responses) 时受支持。

`prompt` 可以是静态对象，也可以是运行时返回对象的函数。关于回调结构，请参见[动态提示词](/openai-agents-js/zh/guides/agents#dynamic-prompts)。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `promptId` | `string` | 提示词的唯一标识符。 |
| `version` | `string` | 要使用的提示词版本。 |
| `variables` | `object` | 用于替换到提示词中的键值对变量。值可为字符串或文本、图像、文件等内容输入类型。 |

任何额外的智能体配置（如 tools 或 instructions）都会覆盖您在存储提示词中配置的值。
