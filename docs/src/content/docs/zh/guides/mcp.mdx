[**Model Context Protocol（MCP）**](https://modelcontextprotocol.io)是一种开放协议，用于标准化应用向 LLM 提供工具和上下文的方式。摘自 MCP 文档：

> MCP 是一种开放协议，用于标准化应用向 LLM 提供上下文的方式。你可以把 MCP 想象成 AI 应用的 USB-C 端口。就像 USB-C 提供了将设备连接到各种外设和配件的标准化方式一样，MCP 也提供了将 AI 模型连接到不同数据源和工具的标准化方式。

该 SDK 支持三种 MCP 服务器：

1. **远程 MCP 服务器工具**——由 [OpenAI Responses API](https://platform.openai.com/docs/guides/tools-remote-mcp) 作为工具使用的远程 MCP 服务器
2. **可流式 HTTP MCP 服务器**——实现了 [可流式 HTTP 传输](https://modelcontextprotocol.io/docs/concepts/transports#streamable-http)的本地或远程服务器
3. **Stdio MCP 服务器**——通过标准输入/输出访问的服务器（最简单的选项）

> 注意：SDK 还包含用于旧版 Server-Sent Events 传输的 `MCPServerSSE`，但 SSE 已被 MCP 项目标记为弃用。新集成建议优先使用可流式 HTTP 或 stdio。

请根据你的使用场景选择服务器类型：

| 你的需求 | 推荐选项 |
| --- | --- |
| 调用可公开访问的远程服务器，并使用默认 OpenAI responses 模型 | **1. 远程 MCP 工具** |
| 使用可公开访问的远程服务器，但在本地触发工具调用 | **2. 可流式 HTTP** |
| 使用本地运行的可流式 HTTP 服务器 | **2. 可流式 HTTP** |
| 将任意可流式 HTTP 服务器用于非 OpenAI-Responses 模型 | **2. 可流式 HTTP** |
| 使用仅支持标准 I/O 协议的本地 MCP 服务器 | **3. Stdio** |

### 托管 MCP 选项参考

`hostedMcpTool(...)` 同时支持 MCP 服务器 URL 和基于连接器的服务器：

| 选项 | 类型 | 说明 |
| --- | --- | --- |
| `serverLabel` | `string` | 必填标签，用于在事件和追踪中标识托管 MCP 服务器。 |
| `serverUrl` | `string` | 远程 MCP 服务器 URL（常规托管 MCP 服务器使用此项）。 |
| `connectorId` | `string` | OpenAI 连接器 id（基于连接器的托管服务器请用它替代 `serverUrl`）。 |
| `authorization` | `string` | 可选的授权令牌，会发送到托管 MCP 后端。 |
| `headers` | `Record<string, string>` | 可选的额外请求头。 |
| `allowedTools` | `string[] \| object` | 暴露给模型的工具名白名单。可传 `string[]` 或 `{ toolNames?: string[] }`。 |
| `deferLoading` | `boolean` | 仅适用于 Responses 的托管 MCP 工具延迟加载。要求同一智能体内包含 `toolSearchTool()`。 |
| `requireApproval` | `'never' \| 'always' \| object` | 托管 MCP 工具调用的审批策略。对象形式可按工具覆盖。默认是 `'never'`。 |
| `onApproval` | Approval callback | 当 `requireApproval` 需要审批处理时，用于程序化批准/拒绝的可选回调。 |

当你希望模型按需通过工具搜索加载托管 MCP 服务器的工具定义，而不是一开始全部暴露时，请设置 `deferLoading: true`。这仅适用于 OpenAI Responses API，且同一请求中需要 `toolSearchTool()`，并建议与 GPT-5.4 及更新的受支持模型版本一起使用。完整的延迟加载配置见[工具指南](/openai-agents-js/guides/tools#deferred-tool-loading-with-tool-search)。

`requireApproval` 的对象形式：

```ts
{
  always?: { toolNames: string[] };
  never?: { toolNames: string[] };
}
```

`onApproval` 签名：

```ts
async function onApproval(
  context,
  item,
): Promise<{
  approve: boolean;
  reason?: string;
}> {}
```

## MCP 服务器生命周期管理

当你使用多个 MCP 服务器时，可以用 `connectMcpServers` 统一连接它们、跟踪失败并集中关闭。该辅助函数会返回一个 `MCPServers` 实例，包含 `active`、`failed` 和 `errors` 集合，这样你可以只把健康服务器传给智能体。

<Code lang="typescript" code={mcpServersExample} title="管理多个 MCP 服务器" />

使用场景：

- **同时使用多个服务器**：并行连接所有服务器，并将 `mcpServers.active` 用于智能体。
- **部分失败处理**：检查 `failed` + `errors`，决定继续还是重试。
- **重试失败服务器**：调用 `mcpServers.reconnect()`（默认仅重试失败服务器）。

如果你需要严格的“全有或全无”连接策略，或不同的超时设置，可使用 `connectMcpServers(servers, options)` 并按你的环境调优选项。

`connectMcpServers` 选项：

| 选项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `connectTimeoutMs` | `number \| null` | `10000` | 每个服务器 `connect()` 的超时。设为 `null` 可禁用。 |
| `closeTimeoutMs` | `number \| null` | `10000` | 每个服务器 `close()` 的超时。设为 `null` 可禁用。 |
| `dropFailed` | `boolean` | `true` | 将失败服务器从 `active` 中排除。 |
| `strict` | `boolean` | `false` | 任一服务器连接失败时抛出错误。 |
| `suppressAbortError` | `boolean` | `true` | 忽略类似 abort 的错误，但仍跟踪失败服务器。 |
| `connectInParallel` | `boolean` | `false` | 并发连接所有服务器，而不是顺序连接。 |

`mcpServers.reconnect(options)` 支持：

| 选项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `failedOnly` | `boolean` | `true` | 仅重试失败服务器（`true`）或重连全部服务器（`false`）。 |

### 异步释放（可选）

如果你的运行时支持 `Symbol.asyncDispose`，`MCPServers` 也支持 `await using` 模式。在 TypeScript 中，请在 `tsconfig.json` 启用 `esnext.disposable`：

```json
{
  "compilerOptions": {
    "lib": ["ES2018", "DOM", "esnext.disposable"]
  }
}
```

然后你可以这样写：

```ts
await using mcpServers = await connectMcpServers(servers);
```

## 其他注意事项

对于**可流式 HTTP**和**Stdio**服务器，每次 `Agent` 运行时都可能调用 `list_tools()` 来发现可用工具。由于这次往返会增加延迟——尤其是远程服务器——你可以在 `MCPServerStdio` 或 `MCPServerStreamableHttp` 中传入 `cacheToolsList: true`，将结果缓存到内存中。

仅在你确信工具列表不会变化时启用此功能。若之后需要使缓存失效，可在服务器实例上调用 `invalidateToolsCache()`。如果你使用 `getAllMcpTools(...)` 的共享 MCP 工具缓存，也可以按服务器名调用 `invalidateServerToolsCache(serverName)` 使缓存失效。

在高级场景下，`getAllMcpTools({ generateMCPToolCacheKey })` 允许你自定义缓存分区方式（例如按服务器 + 智能体 + 运行上下文）。
