护栏可以与您的智能体并行运行，或在其完成前阻塞执行，从而对用户输入或智能体输出执行检查与校验。比如，您可以在调用昂贵模型前先运行一个轻量模型作为护栏。如果护栏检测到恶意使用，它可以触发错误，阻止昂贵模型运行。

护栏分为两类：

1. **输入护栏** 运行在初始用户输入上。
2. **输出护栏** 运行在智能体最终输出上。

## 输入护栏

输入护栏分三步运行：

1. 护栏接收与智能体相同的输入。
2. 护栏函数执行并返回一个包装在[`InputGuardrailResult`](/openai-agents-js/openai/agents/interfaces/inputguardrailresult)中的[`GuardrailFunctionOutput`](/openai-agents-js/openai/agents/interfaces/guardrailfunctionoutput)。
3. 如果 `tripwireTriggered` 为 `true`，则抛出[`InputGuardrailTripwireTriggered`](/openai-agents-js/openai/agents/classes/inputguardrailtripwiretriggered)错误。

> **注意**
> 输入护栏面向用户输入，因此仅当该智能体是工作流中的第一个智能体时才会运行。护栏在智能体本身上进行配置，因为不同智能体通常需要不同的护栏。

### 执行模式

- `runInParallel: true`（默认）与 LLM/工具调用并行启动护栏。这可最小化时延，但若护栏随后触发，模型可能已消耗 token 或已运行工具。
- `runInParallel: false` 在调用模型之前运行护栏，若护栏阻止请求可避免 token 消耗与工具执行。当您更看重安全与成本而非时延时使用此选项。

## 输出护栏

输出护栏分三步运行：

1. 护栏接收由智能体生成的输出。
2. 护栏函数执行并返回一个包装在[`OutputGuardrailResult`](/openai-agents-js/openai/agents/interfaces/outputguardrailresult)中的[`GuardrailFunctionOutput`](/openai-agents-js/openai/agents/interfaces/guardrailfunctionoutput)。
3. 如果 `tripwireTriggered` 为 `true`，则抛出[`OutputGuardrailTripwireTriggered`](/openai-agents-js/openai/agents/classes/outputguardrailtripwiretriggered)错误。

> **注意**
> 仅当智能体是工作流中的最后一个智能体时，输出护栏才会运行。关于实时语音交互，请参见[构建语音智能体](/openai-agents-js/zh/guides/voice-agents/build#guardrails)。

## 工具护栏

工具护栏包装**函数工具**，允许您在执行前后验证或阻止工具调用。它们在工具本身上配置（通过 `tool()` 选项），并在每次工具调用时运行。

- **输入工具护栏** 在工具执行前运行，可用消息拒绝调用或抛出绊线。
- **输出工具护栏** 在工具执行后运行，可用拒绝消息替换输出或抛出绊线。

工具护栏返回一个 `behavior`：

- `allow` — 继续到下一个护栏或工具执行。
- `rejectContent` — 以消息短路（跳过工具调用或替换输出）。
- `throwException` — 立即抛出绊线错误。

工具护栏适用于用 `tool()` 创建的函数工具。托管工具与本地内置工具（`computerTool`、`shellTool`、`applyPatchTool`）不使用此护栏管线。

## 绊线

当护栏失败时，会通过绊线进行信号传递。一旦触发绊线，runner 会抛出相应错误并停止执行。
