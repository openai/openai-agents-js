护栏可以与您的智能体并行运行，或在完成前阻塞执行，使您能够对用户输入或智能体输出进行检查和校验。比如，您可以在调用昂贵模型前先运行一个轻量模型作为护栏。如果护栏检测到恶意使用，它可以触发错误并阻止昂贵模型运行。

护栏有两种类型：

1. **输入护栏** 运行在初始用户输入上。
2. **输出护栏** 运行在最终的智能体输出上。

## 输入护栏

输入护栏分三步运行：

1. 护栏接收传给智能体的同一份输入。
2. 护栏函数执行并返回包裹在[`InputGuardrailResult`](/openai-agents-js/openai/agents/interfaces/inputguardrailresult)中的[`GuardrailFunctionOutput`](/openai-agents-js/openai/agents/interfaces/guardrailfunctionoutput)。
3. 如果 `tripwireTriggered` 为 `true`，会抛出一个[`InputGuardrailTripwireTriggered`](/openai-agents-js/openai/agents/classes/inputguardrailtripwiretriggered)错误。

> **注意**
> 输入护栏用于用户输入，因此它们只会在该智能体是工作流中的*第一个*智能体时运行。护栏配置在智能体本身上，因为不同智能体往往需要不同的护栏。

### 执行模式

- `runInParallel: true`（默认）会在 LLM/工具调用的同时启动护栏。这可最小化时延，但如果护栏随后触发，模型可能已经消耗了 token 或运行了工具。
- `runInParallel: false` 会在调用模型**之前**运行护栏，当护栏阻断请求时，可避免 token 消耗和工具执行。当您更看重安全与成本而非时延时使用此选项。

## 输出护栏

输出护栏分三步运行：

1. 护栏接收由智能体产生的输出。
2. 护栏函数执行并返回包裹在[`OutputGuardrailResult`](/openai-agents-js/openai/agents/interfaces/outputguardrailresult)中的[`GuardrailFunctionOutput`](/openai-agents-js/openai/agents/interfaces/guardrailfunctionoutput)。
3. 如果 `tripwireTriggered` 为 `true`，会抛出一个[`OutputGuardrailTripwireTriggered`](/openai-agents-js/openai/agents/classes/outputguardrailtripwiretriggered)错误。

> **注意**
> 只有当该智能体是工作流中的*最后一个*智能体时才会运行输出护栏。关于实时语音交互，请参阅[构建语音智能体](/openai-agents-js/zh/guides/voice-agents/build#guardrails)。

## 绊线

当护栏失败时，它会通过绊线发出信号。一旦绊线被触发，Runner 会抛出相应的错误并停止执行。
