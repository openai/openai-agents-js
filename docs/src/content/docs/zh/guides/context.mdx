“上下文”一词含义很多。通常有两大类你会关心的上下文：

1. 你的代码在一次运行期间可访问的**本地上下文**：工具所需的依赖或数据、如 `onHandoff` 之类的回调，以及生命周期钩子。
2. 语言模型在生成响应时可见的**智能体/LLM 上下文**。

## 本地上下文

本地上下文由 `RunContext<T>` 类型表示。你可以创建任何对象来保存状态或依赖，并将其传给 `Runner.run()`。所有工具调用和钩子都会接收一个 `RunContext` 包装器，从而读取或修改该对象。

参与同一次运行的每个智能体、工具和钩子都必须使用相同**类型**的上下文。

将本地上下文用于以下场景：

- 关于本次运行的数据（用户名、ID 等）
- 依赖，例如日志记录器或数据获取器
- 辅助函数

<Aside type="note">
  该上下文对象**不会**发送给 LLM。它完全是本地的，你可以自由读取或写入。
</Aside>

## 智能体/LLM 上下文

当调用 LLM 时，它能看到的唯一数据来自会话历史。若要提供额外信息，你有几种选择：

1. 将其添加到智能体的 `instructions`（也称系统或开发者消息）中。它可以是静态字符串，或一个接收上下文并返回字符串的函数。
2. 在调用 `Runner.run()` 时将其包含在 `input` 中。这与 instructions 的方法类似，但允许你将消息放在[指挥链](https://cdn.openai.com/spec/model-spec-2024-05-08.html#follow-the-chain-of-command)中更靠后的位置。
3. 通过函数工具暴露它，以便 LLM 按需获取数据。
4. 使用检索或 Web 搜索工具，将响应锚定到来自文件、数据库或网页的相关数据。
