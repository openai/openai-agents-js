“上下文”（context）是一个多义词。通常可以分为两大类你可能关心的上下文：

1. 你的代码在一次运行期间可访问的**本地上下文**：工具所需的依赖或数据、`onHandoff` 等回调，以及生命周期钩子。
2. 语言模型在生成响应时可见的**智能体/LLM 上下文**。

## 本地上下文

本地上下文由 `RunContext<T>` 类型表示。你可以创建任意对象来保存状态或依赖，并将其传给 `Runner.run()`。所有工具调用和钩子都会接收一个 `RunContext` 包装器，以便读取或修改该对象。

参与同一次运行的每个智能体、工具和钩子都必须使用相同**类型**的上下文。

本地上下文适用于以下内容：

- 关于本次运行的数据（用户姓名、ID 等）
- 日志记录器或数据获取器等依赖
- 帮助函数

<Aside type="note">
  上下文对象**不会**被发送给 LLM。它纯属本地内容，你可以自由读取或写入。
</Aside>

## 智能体/LLM 上下文

当调用 LLM 时，它只能看到会话历史中的数据。若要提供额外信息，你可以：

1. 将其添加到智能体的 `instructions`（也称为 system 或 developer 消息）。这可以是一个静态字符串，或一个接收上下文并返回字符串的函数。
2. 在调用 `Runner.run()` 时将其包含在 `input` 中。这类似于 instructions 技术，但允许你将消息放在[指挥链](https://cdn.openai.com/spec/model-spec-2024-05-08.html#follow-the-chain-of-command)的较低位置。
3. 通过函数工具暴露它，以便 LLM 按需获取数据。
4. 使用检索或 Web 搜索工具，将响应建立在来自文件、数据库或网页的相关数据之上。
