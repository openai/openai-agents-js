Context 是一个含义较多的术语。你可能会关注两大类上下文：

1. 你的代码在一次运行期间可访问的**本地上下文**：工具所需的依赖或数据、`onHandoff` 之类的回调，以及生命周期钩子。
2. 语言模型在生成响应时可见的**智能体/LLM 上下文**。

## 本地上下文

本地上下文由 `RunContext<T>` 类型表示。你可以创建任意对象来保存状态或依赖，并将其传给 `Runner.run()`。所有工具调用和钩子都会收到一个 `RunContext` 包装器，以便读取或修改该对象。

在一次运行中参与的每个智能体、工具和钩子，都必须使用相同的上下文**类型**。

本地上下文可用于以下内容：

- 与运行相关的数据（用户名、ID 等）
- 依赖项，例如日志记录器或数据获取器
- 辅助函数

<Aside type="note">
  上下文对象**不会**发送给 LLM。它完全是本地的，你可以自由读取或写入。
</Aside>

## 智能体/LLM 上下文

调用 LLM 时，它唯一可见的数据来自对话历史。若要让其他信息可用，你有以下几种选择：

1. 将其加入智能体的 `instructions`（也称系统消息或开发者消息）。这可以是静态字符串，也可以是接收上下文并返回字符串的函数。
2. 在调用 `Runner.run()` 时将其放入 `input`。这与 instructions 技术类似，但你可以将消息放在[指令链](https://cdn.openai.com/spec/model-spec-2024-05-08.html#follow-the-chain-of-command)中更靠后的位置。
3. 通过函数工具暴露它，让 LLM 按需获取数据。
4. 使用检索或 Web 搜索工具，使响应基于来自文件、数据库或 Web 的相关数据。
