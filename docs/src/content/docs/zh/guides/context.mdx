“上下文”一词含义很多。你通常会关心两类主要的上下文：

1. 代码在运行期间可访问的**本地上下文**：工具所需的依赖或数据、如 `onHandoff` 之类的回调，以及生命周期钩子。
2. 语言模型在生成响应时可见的**Agent/LLM 上下文**。

## 本地上下文

本地上下文由 `RunContext<T>` 类型表示。你可以创建任意对象来保存状态或依赖，并将其传给 `Runner.run()`。所有工具调用和钩子都会接收一个 `RunContext` 包装器，从而读取或修改该对象。

参与同一次运行的每个智能体、工具和钩子都必须使用相同**类型**的上下文。

将本地上下文用于以下场景：

- 关于本次运行的数据（用户名、ID 等）
- 诸如日志器或数据获取器之类的依赖
- 辅助函数

<Aside type="note">
  上下文对象**不会**发送给 LLM。它纯属本地数据，你可以自由读取或写入。
</Aside>

## Agent/LLM 上下文

当调用 LLM 时，它只能看到来自对话历史的数据。若要提供额外信息，你有几种选择：

1. 添加到 Agent 的 `instructions` 中——也称为 system 或 developer 消息。它可以是静态字符串，或一个接收上下文并返回字符串的函数。
2. 在调用 `Runner.run()` 时将其包含在 `input` 中。这与 instructions 技术类似，但允许你将消息置于[指挥链](https://cdn.openai.com/spec/model-spec-2024-05-08.html#follow-the-chain-of-command)的较低位置。
3. 通过函数工具暴露，让 LLM 按需获取数据。
4. 使用检索或 Web 搜索工具，将响应立足于来自文件、数据库或网页的相关数据。
