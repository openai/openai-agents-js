## 项目设置与凭证

<Steps>

1. **创建项目**

   在本快速开始中，我们将创建一个可在浏览器中使用的语音智能体。如果您想快速搭建一个新项目，可以从 [`Next.js`](https://nextjs.org/docs/getting-started/installation) 或 [`Vite`](https://vite.dev/guide/installation.html) 开始。

   ```bash
   npm create vite@latest my-project -- --template vanilla-ts
   ```

2. **安装推荐包**（需要 Zod v4）

   ```bash
   npm install @openai/agents zod
   ```

3. **生成客户端临时令牌**

   由于该应用将在用户的浏览器中运行，我们需要一种安全的方式，通过 Realtime API 连接到模型。推荐流程与官方的 [使用 WebRTC 的 Realtime API](https://developers.openai.com/api/docs/guides/realtime-webrtc/) 指南一致：您的后端创建一个短期有效的临时客户端令牌，然后浏览器使用该令牌建立 WebRTC 连接。出于测试目的，您也可以使用 `curl` 和常规的 OpenAI API 密钥来生成令牌。

   ```bash
   export OPENAI_API_KEY="sk-proj-...(your own key here)"
   curl -X POST https://api.openai.com/v1/realtime/client_secrets \
      -H "Authorization: Bearer $OPENAI_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "session": {
          "type": "realtime",
          "model": "gpt-realtime-2"
        }
      }'
   ```

   响应中包含一个顶层 `value` 字段，其值以 `ek_` 前缀开头，另外还包含生效的 `session` 对象。建立 WebRTC 连接时，请将 `value` 用作客户端密钥。该令牌的有效期很短，因此后端应在需要时签发新的令牌。如果您的浏览器会话需要带有 `authorization` 或自定义 `headers` 的远程 MCP 服务器工具，请将该托管 MCP 配置包含在发送到 `POST /v1/realtime/client_secrets` 的服务端 `session` 负载中，而不是在浏览器代码中暴露这些凭证。

</Steps>

<Aside type="note">
  生产环境中的浏览器流程：浏览器向您的后端请求一个新的临时令牌，后端使用服务器
  API 密钥调用 `POST /v1/realtime/client_secrets`，然后浏览器将返回的 `ek_...`
  令牌传递给 `session.connect(...)`。
</Aside>

## 语音智能体的创建与连接

<Steps>

1. **创建您的第一个 Agent**

   创建一个新的 [`RealtimeAgent`](/openai-agents-js/openai/agents-realtime/classes/realtimeagent/) 与创建常规的 [`Agent`](/openai-agents-js/zh/guides/agents) 非常相似。

   ```typescript
   import { RealtimeAgent } from '@openai/agents/realtime';

   const agent = new RealtimeAgent({
     name: 'Assistant',
     instructions: 'You are a helpful assistant.',
   });
   ```

2. **创建会话**

   与常规智能体不同，语音智能体会持续运行在 `RealtimeSession` 中，随着时间推移处理对话以及与模型的连接。该会话还管理音频处理、中断以及更广泛的对话生命周期，您稍后将对其进行配置。

   ```typescript
   import { RealtimeSession } from '@openai/agents/realtime';

   const session = new RealtimeSession(agent, {
     model: 'gpt-realtime-2',
   });
   ```

   `RealtimeSession` 构造函数将 `agent` 作为第一个参数。该智能体将是用户首先与之交互的智能体。

3. **连接到会话**

   要连接到会话，您需要传入之前生成的客户端临时令牌。

   ```typescript
   await session.connect({ apiKey: 'ek_...(put your own key here)' });
   ```

   在浏览器中，这会使用 WebRTC 连接到 Realtime API，并自动为您配置麦克风采集和音频播放。在默认的 WebRTC 路径下，SDK 会在数据通道打开后立即发送初始会话配置，并尝试在 `connect()` 返回前等待匹配的 `session.updated` 确认；如果该确认始终未到达，则会在超时后继续返回。如果您在 Node.js 等服务器运行时中运行 `RealtimeSession`，SDK 会自动回退到 WebSocket；在 WebSocket 路径下，`connect()` 会在套接字打开且初始配置已发送后返回，因此 `session.updated` 可能会稍晚到达。您可以在[传输机制](/openai-agents-js/zh/guides/voice-agents/transport)指南中了解更多关于传输选择的信息。

</Steps>

<Aside type="note">
  如果您希望对对等连接或原始传输事件进行更底层的控制，请参阅
  [`examples/realtime-next` 中的 `/raw-client` 示例
  ](https://github.com/openai/openai-agents-js/tree/main/examples/realtime-next)。
</Aside>
