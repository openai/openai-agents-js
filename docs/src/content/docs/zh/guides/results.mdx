当您[运行智能体](/openai-agents-js/zh/guides/running-agents)时，您会收到以下之一：

- 如果调用 `run` 时未设置 `stream: true`，则返回 [`RunResult`](/openai-agents-js/openai/agents/classes/runresult)
- 如果调用 `run` 时设置了 `stream: true`，则返回 [`StreamedRunResult`](/openai-agents-js/openai/agents/classes/streamedrunresult)。关于流式传输的详情，也可参阅[流式传输指南](/openai-agents-js/zh/guides/streaming)。

这两种结果类型都暴露相同的核心结果视图，例如 `finalOutput`、`newItems`、`interruptions` 和 `state`。`StreamedRunResult` 额外提供流式控制能力，例如 `completed`、`toStream()`、`toTextStream()` 和 `currentAgent`。

## 结果视图选择

大多数应用只需要少量属性：

| 如果您需要…… | 使用 |
| --- | --- |
| 展示给用户的最终答案 | `finalOutput` |
| 可重放的下一轮输入（包含完整本地转录） | `history` |
| 仅本次运行新生成的、模型形状的条目 | `output` |
| 包含智能体/工具/交接元数据的丰富运行条目 | `newItems` |
| 通常应处理下一轮用户输入的智能体 | `lastAgent` 或 `activeAgent` |
| OpenAI Responses API 基于 `previousResponseId` 的串联 | `lastResponseId` |
| 待审批项与可恢复快照 | `interruptions` 和 `state` |
| 应用上下文、审批、用量和嵌套智能体工具输入 | `runContext` |
| 当前嵌套 `Agent.asTool()` 调用的元数据，例如在 `customOutputExtractor` 内 | `agentToolInvocation` |
| 原始模型调用或护栏诊断信息 | `rawResponses` 和护栏结果数组 |

## 最终输出

`finalOutput` 属性包含最后一个运行的智能体的最终输出。该结果可能是：

- `string`——任何未定义 `outputType` 的智能体默认类型
- `unknown`——当智能体将 JSON schema 定义为输出类型时。在这种情况下 JSON 已被解析，但您仍需手动校验其类型。
- `z.infer<outputType>`——当智能体将 Zod schema 定义为输出类型时。输出会自动按该 schema 解析。
- `undefined`——当智能体未产出输出（例如在产出前已停止）时

在流式运行尚未完成，或运行在到达最终输出前因审批中断而暂停时，`finalOutput` 也会是 `undefined`。

如果您在交接中使用了不同输出类型，应使用 `Agent.create()` 方法而不是 `new Agent()` 构造函数来创建智能体。

这会让 SDK 能推断所有可能交接路径上的输出类型，并为 `finalOutput` 属性提供联合类型。

## 输入与输出视图

这些属性回答的是不同问题：

| 属性 | 包含内容 | 最适用场景 |
| --- | --- | --- |
| `input` | 本次运行的基础输入。若交接输入过滤器重写了历史，这里反映的是运行继续使用的过滤后输入。 | 审计本次运行实际使用了什么输入 |
| `output` | 仅包含本次运行生成的模型形状条目，不含智能体元数据。 | 只存储或重放新的模型增量 |
| `newItems` | 带智能体/工具/交接元数据的丰富 [`RunItem`](/openai-agents-js/openai/agents/type-aliases/runitem) 包装项。 | 日志、UI、审计与调试 |
| `history` | 基于 `input + newItems` 构建的可重放下一轮输入。 | 手动聊天循环与客户端管理会话状态 |

在实践中：

- 当您在应用里手动维护完整会话时，使用 `history`。
- 当您已在其他地方保存历史，只想拿到本次新生成条目时，使用 `output`。
- 当您需要智能体关联、工具输出、交接边界或审批条目时，使用 `newItems`。
- 如果您使用 `conversationId` 或 `previousResponseId`，通常不需要把 `history` 再传回 `run()`。而是只传新的用户输入，并复用服务器管理的 ID。完整对比见[运行智能体](/openai-agents-js/zh/guides/running-agents#state-and-conversation-management)。

## 新条目

`newItems` 能为您提供本次运行过程的最完整视图。常见条目类型有：

- 用于助手消息的 [`RunMessageOutputItem`](/openai-agents-js/openai/agents/classes/runmessageoutputitem)。
- 用于推理条目的 [`RunReasoningItem`](/openai-agents-js/openai/agents/classes/runreasoningitem)。
- 用于 Responses 工具搜索请求及其返回的已加载工具定义的 [`RunToolSearchCallItem`](/openai-agents-js/openai/agents/classes/runtoolsearchcallitem) 和 [`RunToolSearchOutputItem`](/openai-agents-js/openai/agents/classes/runtoolsearchoutputitem)。
- 用于工具调用及其结果的 [`RunToolCallItem`](/openai-agents-js/openai/agents/classes/runtoolcallitem) 和 [`RunToolCallOutputItem`](/openai-agents-js/openai/agents/classes/runtoolcalloutputitem)。
- 用于因审批而暂停的工具调用的 [`RunToolApprovalItem`](/openai-agents-js/openai/agents/classes/runtoolapprovalitem)。
- 用于交接请求与已完成转移的 [`RunHandoffCallItem`](/openai-agents-js/openai/agents/classes/runhandoffcallitem) 和 [`RunHandoffOutputItem`](/openai-agents-js/openai/agents/classes/runhandoffoutputitem)。

当您需要知道某个条目由哪个智能体产生，或它是否标记了工具、工具搜索、交接或审批边界时，应优先使用 `newItems` 而非 `output`。当您使用 `toolSearchTool()` 时，这些工具搜索条目是查看在常规工具调用前加载了哪些延迟工具或命名空间的最直接方式。

## 会话续接与恢复

### 活跃智能体

`lastAgent` 属性包含最后一个运行的智能体。在交接后，这通常是下一轮用户输入应复用的最佳智能体。`activeAgent` 是该值的别名。

在流式模式下，`currentAgent` 会在运行仍在进行时告知您当前活跃的智能体。

### 中断与可恢复状态

如果某个工具需要审批，运行会暂停，`interruptions` 中会包含待处理的 [`RunToolApprovalItem`](/openai-agents-js/openai/agents/classes/runtoolapprovalitem)。这些审批可能来自直接工具、交接后到达的工具，或嵌套的 `agent.asTool()` 运行。

通过 `result.state.approve(...)` / `result.state.reject(...)` 处理审批后，将同一个 `state` 传回 `run()` 即可恢复。您不需要一次处理完所有中断。如果您只处理了部分条目就重新运行，已处理的调用会继续执行，未处理的仍保持待处理，并会再次使运行暂停。

`state` 属性是结果背后的可序列化快照。它可用于[人机协作](/openai-agents-js/zh/guides/human-in-the-loop)、重试流程，或任何需要稍后恢复暂停运行的场景。

### 服务器管理的续接

当您使用 OpenAI Responses API 串联时，`lastResponseId` 是下一轮要作为 `previousResponseId` 传入的值。

如果您已经通过 `history`、`session` 或 `conversationId` 续接会话，通常不需要 `lastResponseId`。如果您需要多步骤运行中的每次原始模型响应，请改为查看 `rawResponses`。

## 嵌套智能体工具元数据

`agentToolInvocation` 用于嵌套 `Agent.asTool()` 的结果，特别是在 `customOutputExtractor` 内需要当前工具调用元数据时。它不是一个通用的“整个运行已完成”汇总字段。

在该嵌套上下文中，`agentToolInvocation` 暴露：

- `toolName`
- `toolCallId`
- `toolArguments`

如果您还需要传入该嵌套智能体工具运行的结构化输入，可与 `result.runContext.toolInput` 配合使用。

在普通顶层 `run()` 结果中，这个字段通常是 `undefined`。该元数据仅在运行时存在，不会序列化进 `RunState`。相关模式见[Agents as tools](/openai-agents-js/zh/guides/tools#4-agents-as-tools)。

## 流式结果

`StreamedRunResult` 继承了上述相同结果视图，但额外提供流式专用控制能力：

- `toTextStream()`：仅输出助手文本。
- `toStream()` 或 `for await ... of stream`：获取完整事件流。
- `completed`：等待运行及所有后处理回调完成。
- `error` 和 `cancelled`：检查流式终态。
- `currentAgent`：在运行中跟踪当前活跃智能体。

如果您需要流式运行稳定后的最终状态，请在读取 `finalOutput`、`history`、`interruptions` 或其他汇总属性前先等待 `completed`。若要逐事件处理，请参阅[流式传输指南](/openai-agents-js/zh/guides/streaming)。

## 诊断与高级字段

### 运行上下文

`runContext` 属性是在结果上公开支持的运行上下文视图。`result.runContext.context` 是您的应用上下文；同一对象也携带 SDK 管理的运行时元数据，例如审批、用量和嵌套 `toolInput`。完整结构见[上下文管理](/openai-agents-js/zh/guides/context)。

### 原始响应

`rawResponses` 包含运行期间收集到的原始模型响应。多步骤运行可能产生不止一个响应，例如跨交接或重复的工具/模型循环。

### 护栏结果

`inputGuardrailResults` 和 `outputGuardrailResults` 属性包含智能体级护栏结果。工具级护栏结果则通过 `toolInputGuardrailResults` 和 `toolOutputGuardrailResults` 单独提供。

当您希望记录护栏决策、检查护栏函数返回的额外元数据，或调试某次运行为何被拦截时，可使用这些数组。

### 用量

Token 用量会聚合在 `result.state.usage` 中，它会跟踪本次运行的请求次数和 token 总量。同一用量对象也可通过 `result.runContext.usage` 获取。对于流式运行，这些数据会随响应到达而更新。
