使用 `setOpenAIResponsesTransport('websocket')` 启用 WebSocket 传输机制，使用 `setOpenAIResponsesTransport('http')` 切换回 HTTP。如果您通过代理或网关路由 WebSocket 流量，请设置 `OPENAI_WEBSOCKET_BASE_URL`（或在 `OpenAIProvider` 上配置 `websocketBaseURL`）。

这个进程级默认设置只会影响之后通过默认 OpenAI 提供商解析的模型。如果您传入具体的 `Model` 实例或自定义 `modelProvider`，请改为在那里配置传输机制。请参阅[模型](/openai-agents-js/zh/guides/models#responses-websocket-transport)。

### 调试日志

SDK 使用 [`debug`](https://www.npmjs.com/package/debug) 包记录调试日志。将 `DEBUG` 环境变量设置为 `openai-agents*`，即可查看详细日志。

```bash
export DEBUG=openai-agents*
```

要记录会话持久化活动，请设置 `OPENAI_AGENTS__DEBUG_SAVE_SESSION=1`。

您可以使用 `@openai/agents` 中的 `getLogger(namespace)`，为自己的模块获取带命名空间的日志记录器。

#### 日志中的敏感数据

某些日志可能包含用户数据。可通过设置以下环境变量将其禁用。

要禁用 LLM 输入和输出日志：

```bash
export OPENAI_AGENTS_DONT_LOG_MODEL_DATA=1
```

这还会从 SDK 错误日志中隐去与模型相关的错误对象和详细信息，仅保留安全的错误类型。

要禁用工具输入和输出日志：

```bash
export OPENAI_AGENTS_DONT_LOG_TOOL_DATA=1
```

这还会从 SDK 错误日志中隐去与工具相关的错误对象和详细信息，包括工具执行、审批格式化以及实时工具处理失败的信息。
