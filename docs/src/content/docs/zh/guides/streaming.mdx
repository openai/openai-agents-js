Agents SDK 可以逐步输出模型以及其他执行步骤的结果。流式传输可让您的 UI 保持响应，而无需等待整个最终结果再更新用户。

## 启用流式传输

向 `Runner.run()` 传入 `{ stream: true }` 选项以获取一个流式对象，而不是完整结果：

<Code lang="typescript" code={basicStreamingExample} title="启用流式传输" />

启用流式传输后，返回的 `stream` 实现了 `AsyncIterable` 接口。每个产出的事件都是一个描述运行过程中发生了什么的对象。该流会产生三种事件类型之一，分别描述智能体执行的不同部分。大多数应用只需要模型的文本，因此流提供了辅助方法。

### 获取文本输出

调用 `stream.toTextStream()` 以获取已发出的文本流。当 `compatibleWithNodeStreams` 为 `true` 时，返回值为常规的 Node.js `Readable`。我们可以将其直接管道传输到 `process.stdout` 或其他目标。

`stream.completed` 这个 promise 会在运行和所有待处理回调完成后解析。如果您想确保没有更多输出，请务必等待它完成。

## 事件类型

该流会产生三种不同的事件类型：

### raw_model_stream_event

```ts
type RunRawModelStreamEvent = {
  type: 'raw_model_stream_event';
  data: ResponseStreamEvent;
};
```

示例：

```json
{
  "type": "raw_model_stream_event",
  "data": {
    "type": "output_text_delta",
    "delta": "Hello"
  }
}
```

### run_item_stream_event

```ts
type RunItemStreamEvent = {
  type: 'run_item_stream_event';
  name: RunItemStreamEventName;
  item: RunItem;
};
```

交接负载示例：

```json
{
  "type": "run_item_stream_event",
  "name": "handoff_occurred",
  "item": {
    "type": "handoff_call",
    "id": "h1",
    "status": "completed",
    "name": "transfer_to_refund_agent"
  }
}
```

### agent_updated_stream_event

```ts
type RunAgentUpdatedStreamEvent = {
  type: 'agent_updated_stream_event';
  agent: Agent<any, any>;
};
```

示例：

```json
{
  "type": "agent_updated_stream_event",
  "agent": {
    "name": "Refund Agent"
  }
}
```

## 流式传输中的人工干预

流式传输与会暂停执行的交接兼容（例如工具需要审批时）。流对象上的 `interruption` 字段可暴露这些中断，您可以为每个中断调用 `state.approve()` 或 `state.reject()` 以继续执行。再次以 `{ stream: true }` 执行可恢复流式输出。

## 提示

- 在退出前记得等待 `stream.completed`，以确保所有输出已刷新。
- 初始的 `{ stream: true }` 选项仅适用于提供它的那次调用。如果您使用 `RunState` 重新运行，必须再次指定该选项。
- 如果您的应用只关心文本结果，优先使用 `toTextStream()`，以避免处理单个事件对象。

借助流式传输和事件系统，您可以将智能体集成到聊天界面、终端应用程序，或任何用户能从增量更新中受益的场景。
