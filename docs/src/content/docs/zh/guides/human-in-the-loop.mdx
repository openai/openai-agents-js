本指南演示如何使用 SDK 中内置的人工干预支持，在有人介入时暂停并恢复智能体运行。

当前的主要用例是对敏感的工具执行请求审批。

### 流程

1. 如果智能体决定调用一个（或多个）工具，它会通过评估 `needsApproval` 来检查该工具是否需要审批。
2. 如果需要审批，智能体会检查审批是否已被授予或被拒绝。
   - 如果尚未批准或拒绝，工具会向智能体返回一条静态消息，表示该工具调用无法执行。
   - 如果缺少批准/拒绝，它将触发工具审批请求。
3. 智能体会收集所有工具审批请求并中断执行。
4. 如果有任何中断，[执行结果](/openai-agents-js/zh/guides/results) 中将包含一个描述待处理步骤的 `interruptions` 数组。当工具调用需要确认时，会出现 `type: "tool_approval_item"` 的 `ToolApprovalItem`。
5. 你可以调用 `result.state.approve(interruption)` 或 `result.state.reject(interruption)` 来批准或拒绝该工具调用。
6. 处理完所有中断后，你可以通过将 `result.state` 传回 `runner.run(agent, state)` 来恢复执行，其中 `agent` 是触发整体运行的原始智能体。
7. 流程从步骤 1 重新开始。

## 处理较长审批时间

人工干预流程被设计为可在较长时间内中断，而无需让你的服务器持续运行。如果你需要先结束请求并稍后继续，可以序列化状态并在之后恢复。

你可以使用 `JSON.stringify(result.state)` 序列化状态，稍后将序列化的状态传入 `RunState.fromString(agent, serializedState)` 即可恢复，其中 `agent` 是触发整体运行的智能体实例。

这样你可以将序列化后的状态存储在数据库中，或与请求一起保存。

### 待处理任务的版本管理

<Aside>
  这主要适用于你在对智能体进行更改的同时，需要长时间存储序列化状态的情况。
</Aside>

如果审批请求耗时较长，并且你打算以有意义的方式对智能体定义进行版本化或升级你的 Agents SDK 版本，我们目前建议你通过使用包别名并行安装两个版本的 Agents SDK，来实现你自己的分支逻辑。

在实践中，这意味着给你自己的代码分配一个版本号，并将其与序列化状态一起存储，并在反序列化时引导到正确的代码版本。
