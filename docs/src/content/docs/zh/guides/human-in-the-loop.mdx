本指南演示如何使用 SDK 内置的人工干预支持，根据人工介入暂停与恢复智能体运行。

当前的主要用例是对敏感工具执行进行审批。

### 流程

1. 如果智能体决定调用一个或多个工具，它会通过评估 `needsApproval` 来检查该工具是否需要审批。
2. 如果需要审批，智能体会检查审批是否已经被授予或被拒绝。
   - 如果审批尚未被授予或拒绝，工具将向智能体返回一条静态消息，说明该工具调用无法执行。
   - 如果审批/拒绝缺失，将触发一个工具审批请求。
3. 智能体会收集所有工具审批请求并中断执行。
4. 如果有中断，[执行结果](/openai-agents-js/zh/guides/results) 中将包含一个描述待处理步骤的 `interruptions` 数组。当工具调用需要确认时，会出现一个 `type: "tool_approval_item"` 的 `ToolApprovalItem`。
5. 您可以调用 `result.state.approve(interruption)` 或 `result.state.reject(interruption)` 来批准或拒绝该工具调用。
6. 处理完所有中断后，您可以将 `result.state` 传回 `runner.run(agent, state)` 以恢复执行，其中 `agent` 是触发整个运行的原始智能体。
7. 流程从第 1 步重新开始。

## 处理较长审批时间

人工干预流程被设计为可在较长时间内中断，而无需保持服务器运行。如果您需要结束该请求并稍后继续，可以序列化状态并在之后恢复。

您可以使用 `JSON.stringify(result.state)` 序列化状态，稍后将序列化状态传入 `RunState.fromString(agent, serializedState)` 来恢复，其中 `agent` 是触发整个运行的智能体实例。

这样，您可以将序列化状态存储在数据库中，或与请求一同存储。

### 待处理任务的版本管理

<Aside>
  这主要适用于当您在对智能体进行修改的同时，需将序列化状态保存较长时间的情形。
</Aside>

如果审批请求需要较长时间，并且您打算以有意义的方式为智能体定义做版本化，或升级您的 Agents SDK 版本，我们目前建议您通过使用包别名并行安装两个版本的 Agents SDK，来实现自定义的分支逻辑。

实践中，这意味着为您的代码分配版本号，并将其与序列化状态一并存储，同时在反序列化时引导到您代码的正确版本。
