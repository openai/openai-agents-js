本指南演示如何使用 SDK 内置的人工干预（human-in-the-loop）支持，在需要人工介入时暂停并恢复智能体运行。

### 流程

1. 当智能体决定调用一个或多个工具时，会通过评估 `needsApproval` 检查该工具是否需要审批。
2. 如果需要审批，智能体会检查审批是否已经被授予或被拒绝。
   - 如果尚未批准或拒绝，工具会向智能体返回一条静态消息，说明该工具调用无法执行。
   - 如果缺少批准/拒绝，将触发工具审批请求。
3. 智能体会收集所有工具审批请求并中断执行。
4. 如果有任何中断，[执行结果](/openai-agents-js/zh/guides/results) 将包含一个描述待处理步骤的 `interruptions` 数组。当某个工具调用需要确认时，会出现一个带有 `type: "tool_approval_item"` 的 `ToolApprovalItem`。
5. 你可以调用 `result.state.approve(interruption)` 或 `result.state.reject(interruption)` 来批准或拒绝该工具调用。
6. 处理完所有中断后，你可以通过将 `result.state` 传回 `runner.run(agent, state)` 来恢复执行，其中 `agent` 是触发整体运行的原始智能体。
7. 流程从第 1 步重新开始。

## 处理较长审批时间

人工干预流程被设计为可中断较长时间而无需保持你的服务器持续运行。如果你需要中断请求并在稍后继续，可以序列化状态并在之后恢复。

你可以使用 `JSON.stringify(result.state)` 序列化状态，并在稍后将序列化的状态传入 `RunState.fromString(agent, serializedState)` 来恢复，其中 `agent` 是触发整体运行的智能体实例。

这样你就可以将序列化状态存储在数据库中，或与请求一起存储。

### 待处理任务的版本管理

<Aside>
  这主要适用于你在对智能体进行变更的同时，计划将序列化状态长时间存储的情况。
</Aside>

如果审批请求耗时较长，并且你打算以有意义的方式对智能体定义进行版本化或升级你的 Agents SDK 版本，我们目前建议你通过使用包别名并行安装两个 Agents SDK 版本，自行实现分支逻辑。

在实践中，这意味着给你的代码分配一个版本号，并将其与序列化状态一起存储，同时在反序列化时引导到你代码的正确版本。
