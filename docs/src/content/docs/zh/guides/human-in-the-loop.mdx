### 流程

1. 如果智能体决定调用一个或多个工具，会通过评估 `needsApproval` 检查该工具是否需要审批。
2. 如果需要审批，智能体会检查该调用是否已被批准或拒绝。
   - 如果尚未被批准或拒绝，工具会向智能体返回一条静态消息，说明该工具调用无法执行。
   - 如果审批／拒绝缺失，将触发一个工具审批请求。
3. 智能体会收集所有工具审批请求并中断执行。
4. 如果存在中断，[执行结果](/openai-agents-js/zh/guides/results) 将包含一个 `interruptions` 数组描述待处理步骤。当某个工具调用需要确认时，会出现一个 `type: "tool_approval_item"` 的 `ToolApprovalItem`。
5. 您可以调用 `result.state.approve(interruption)` 或 `result.state.reject(interruption)` 来批准或拒绝该工具调用。
6. 处理完所有中断后，可将 `result.state` 传回 `runner.run(agent, state)` 以恢复执行，其中 `agent` 是触发整个运行的原始智能体。
7. 流程从步骤 1 重新开始。

## 处理较长审批时延

人工干预流程被设计为可长时间中断，而无需持续运行您的服务器。如果需要先结束请求，稍后再继续，您可以序列化状态并在之后恢复。

您可以使用 `JSON.stringify(result.state)` 序列化状态，并在稍后将序列化后的状态传入 `RunState.fromString(agent, serializedState)` 来恢复，其中 `agent` 是触发整个运行的智能体实例。

这样您可以将序列化后的状态存储在数据库中，或与您的请求一并保存。

### 待办任务的版本管理

<Aside>
  当您在对智能体进行改动的同时，需要长时间存储序列化状态时，这一点尤为适用。
</Aside>

如果审批请求耗时较长，并且您计划以有意义的方式为智能体定义做版本化，或升级您的 Agents SDK 版本，我们目前建议通过包别名并行安装两个版本的 Agents SDK，自行实现分支逻辑。

在实践中，这意味着为您的代码分配一个版本号，并将其与序列化状态一同存储，并在反序列化时引导至正确的代码版本。
