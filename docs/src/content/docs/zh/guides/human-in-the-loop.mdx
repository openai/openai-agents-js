本指南演示如何使用 SDK 内置的人工干预（Human in the loop）支持，根据人工介入暂停和恢复智能体运行。

目前的主要用例是对敏感的工具执行请求审批。

### 流程

1. 如果智能体决定调用一个（或多个）工具，它会通过评估 `needsApproval` 来检查该工具是否需要审批。
2. 如果需要审批，智能体会检查审批是否已被授予或拒绝。
   - 如果审批尚未授予或拒绝，该工具会向智能体返回一条静态消息，说明无法执行该工具调用。
   - 如果审批/拒绝缺失，这将触发一个工具审批请求。
3. 智能体会收集所有工具审批请求并中断执行。
4. 如果有任何中断，[执行结果](/openai-agents-js/zh/guides/results) 中将包含一个 `interruptions` 数组，描述待处理步骤。当某个工具调用需要确认时，会出现一个 `type: "tool_approval_item"` 的 `ToolApprovalItem`。
5. 您可以调用 `result.state.approve(interruption)` 或 `result.state.reject(interruption)` 来批准或拒绝该工具调用。
6. 处理完所有中断后，您可以将 `result.state` 传回 `runner.run(agent, state)` 来恢复执行，其中 `agent` 是触发整体运行的原始智能体。
7. 流程从第 1 步重新开始。

## 处理较长审批时间

人工干预流程被设计为可中断较长时间而无需保持服务器运行。如果您需要中止请求并在之后继续，可以序列化状态后再恢复。

您可以使用 `JSON.stringify(result.state)` 序列化状态，并在稍后将序列化的状态传入 `RunState.fromString(agent, serializedState)` 以恢复，其中 `agent` 是触发整体运行的智能体实例。

这样，您可以将序列化状态存储在数据库中，或与您的请求一并保存。

### 待处理任务的版本管理

<Aside>
  这主要适用于在对智能体进行更改的同时，打算长期存储序列化状态的情况。
</Aside>

如果您的审批请求需要更长时间，并且您打算以有意义的方式对智能体定义进行版本化或升级 Agents SDK 版本，我们目前建议您使用包别名并行安装两个版本的 Agents SDK，自行实现分支逻辑。

在实践中，这意味着为您的代码分配一个版本号，并将其与序列化状态一起存储，同时在反序列化时引导到正确的代码版本。
