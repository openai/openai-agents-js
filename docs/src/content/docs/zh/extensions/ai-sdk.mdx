Agents SDK 开箱即用，可通过 Responses API 或 Chat Completions API 搭配 OpenAI 模型使用。不过，如果您希望使用其他模型，[Vercel AI SDK](https://sdk.vercel.ai/) 提供了一系列受支持的模型，可通过此适配器接入 Agents SDK。

## 设置

<Steps>

1. 通过安装 extensions 包来安装 AI SDK 适配器：

   ```bash
   npm install @openai/agents-extensions
   ```

2. 从 [Vercel 的 AI SDK](https://ai-sdk.dev/docs/foundations/providers-and-models) 选择所需的模型包并安装：

   ```bash
   npm install @ai-sdk/openai
   ```

## 选择合适的集成

`@openai/agents-extensions` 中有两个相关集成：

- `@openai/agents-extensions/ai-sdk`：适配 AI SDK 模型，使 `Agent` 可在其上运行。
- `@openai/agents-extensions/ai-sdk-ui`：适配流式 Agents SDK 运行结果，使 AI SDK UI 路由可返回标准流式 `Response`。

### AI SDK 模型说明

- `@openai/agents-extensions/ai-sdk` 适配器仍处于 beta 阶段，因此值得针对您选择的提供方（尤其是规模较小的提供方）进行仔细测试。
- 如果您使用的是 OpenAI 模型，建议优先使用默认的 OpenAI 模型提供方，而不是此适配器。
- 受支持的 AI SDK 提供方必须暴露 `specificationVersion` `v2` 或 `v3`。如果您需要旧版 v1 提供方风格，请将 [examples/ai-sdk-v1](https://github.com/openai/openai-agents-js/tree/main/examples/ai-sdk-v1) 中的模块复制到您的项目中。
- 这里不支持 Deferred Responses 的工具延迟加载流程。这包括 `toolNamespace()`、设置了 `deferLoading: true` 的函数工具，以及 `toolSearchTool()`。如果您需要工具搜索，请直接使用 OpenAI Responses 模型。请参阅 [Tools guide](/openai-agents-js/zh/guides/tools#deferred-tool-loading-with-tool-search) 和 [Models guide](/openai-agents-js/zh/guides/models#responses-only-deferred-tool-loading)。

## AI SDK UI 流辅助函数

`@openai/agents-extensions/ai-sdk-ui` 提供了响应辅助函数，用于将 Agents SDK 流接入 AI SDK UI 路由：

- `createAiSdkTextStreamResponse(source, options?)`：用于纯文本流式响应。
- `createAiSdkUiMessageStreamResponse(source, options?)`：用于 `UIMessageChunk` 流式响应。

这两个辅助函数都接受 `StreamedRunResult`、类流 source 或兼容的包装对象，并返回一个带有流式友好头部的 `Response`。

当您的 UI 需要工具调用或推理片段等结构化分块时，使用 `createAiSdkUiMessageStreamResponse(...)`。当您只需要纯文本时，使用 `createAiSdkTextStreamResponse(...)`。
