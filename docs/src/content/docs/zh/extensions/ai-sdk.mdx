Agents SDK 开箱即用地支持通过 Responses API 或 Chat Completions API 使用 OpenAI 模型。不过，如果您想使用其他模型，[Vercel AI SDK](https://sdk.vercel.ai/) 提供了一系列受支持的模型，可通过此适配器接入 Agents SDK。

## 设置

<Steps>

1. 安装 extensions 包，以安装 AI SDK 适配器：

   ```bash
   npm install @openai/agents-extensions
   ```

2. 从 [Vercel 的 AI SDK](https://ai-sdk.dev/docs/foundations/providers-and-models) 中选择您想要的模型包并安装：

   ```bash
   npm install @ai-sdk/openai
   ```

## 重试

`modelSettings.retry` 同样适用于由 AI SDK 支持的模型，因为重试是由 Agents 运行时实现的，而不只是由默认的 OpenAI 提供方实现。

这意味着您可以附加与其他地方相同的重试配置：

- 在 `Agent`、`Runner` 或两者上设置 `modelSettings.retry`。
- 组合 `retryPolicies`，例如 `networkError()`、`httpStatus([...])` 或 `providerSuggested()`。
- 请注意，只有当被包装的 AI SDK 模型能够通过适配器提供重试建议时，`providerSuggested()` 才会有帮助。

有关使用 `aisdk(openai(...))` 的完整示例，请参见 [`examples/ai-sdk/retry.ts`](https://github.com/openai/openai-agents-js/tree/main/examples/ai-sdk/retry.ts)。有关重试 API 本身（包括流式传输和有状态后续请求的安全边界），请参见[模型](/openai-agents-js/zh/guides/models#model-retries)。

## 选择合适的集成

`@openai/agents-extensions` 中有两个相关的集成：

- `@openai/agents-extensions/ai-sdk` 会适配 AI SDK 模型，以便 `Agent` 可以在其上运行。
- `@openai/agents-extensions/ai-sdk-ui` 会适配流式的 Agents SDK 运行结果，以便 AI SDK UI 路由可以返回标准的流式 `Response`。

### AI SDK 模型说明

- `@openai/agents-extensions/ai-sdk` 适配器仍处于 beta 阶段，因此值得结合您选择的提供方仔细测试，尤其是较小的提供方。
- 如果您使用的是 OpenAI 模型，优先选择默认的 OpenAI 模型提供方，而不是此适配器。
- 支持的 AI SDK 提供方必须暴露 `specificationVersion` `v2` 或 `v3`。如果您需要较旧的 v1 提供方形式，请将 [examples/ai-sdk-v1](https://github.com/openai/openai-agents-js/tree/main/examples/ai-sdk-v1) 中的模块复制到您的项目中。
- 通过此适配器使用时，计算机工具需要显示元数据。请确保该工具同时包含 `environment` 和 `dimensions` 元数据。
- 此处不支持 Deferred Responses 工具加载流程。这包括 `toolNamespace()`、带有 `deferLoading: true` 的函数工具以及 `toolSearchTool()`。如果您需要工具搜索，请直接使用 OpenAI Responses 模型。请参见[工具](/openai-agents-js/zh/guides/tools#deferred-tool-loading-with-tool-search)和[模型](/openai-agents-js/zh/guides/models#responses-only-deferred-tool-loading)。

## AI SDK UI 流辅助函数

`@openai/agents-extensions/ai-sdk-ui` 提供了响应辅助函数，用于将 Agents SDK 流接入 AI SDK UI 路由：

- `createAiSdkTextStreamResponse(source, options?)`：用于纯文本流式响应。
- `createAiSdkUiMessageStreamResponse(source, options?)`：用于 `UIMessageChunk` 流式响应。

这两个辅助函数都接受 `StreamedRunResult`、类流式源对象或兼容的包装对象，并返回一个带有流式友好标头的 `Response`。

当您的 UI 需要结构化分块（例如工具调用或推理片段）时，请使用 `createAiSdkUiMessageStreamResponse(...)`。当您只需要纯文本时，请使用 `createAiSdkTextStreamResponse(...)`。

这两个辅助函数还可通过 `options` 接受可选的响应设置：

- `headers`：要合并到流式响应中的额外响应头。
- `status`：返回的 `Response` 的 HTTP 状态码。
- `statusText`：返回的 `Response` 的 HTTP 状态文本。
