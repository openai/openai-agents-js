Agents SDK 开箱即用即可通过 Responses API 或 Chat Completions API 搭配 OpenAI 模型。不过，如果您想使用其他模型，[Vercel AI SDK](https://sdk.vercel.ai/) 提供了一系列受支持的模型，可通过此适配器接入 Agents SDK。

## 设置

<Steps>

1. 通过安装 extensions 包来安装 AI SDK 适配器：

   ```bash
   npm install @openai/agents-extensions
   ```

2. 从 [Vercel 的 AI SDK](https://ai-sdk.dev/docs/foundations/providers-and-models) 中选择所需的模型包并安装：

   ```bash
   npm install @ai-sdk/openai
   ```

某些提供商会将 structured outputs 以带有额外包装的纯文本返回，例如 JSON 代码围栏。如果您需要在 Agents 运行时校验最终输出前进行提供商特定清理，请在创建适配器时传入 `transformOutputText`：

`transformOutputText` 会在非流式响应的最终 assistant 文本上运行，也会在流式响应的最终 `response_done` 事件上运行。它不会修改增量 `output_text_delta` 事件。

## 集成方案选择

`@openai/agents-extensions` 中有两个相关集成：

- `@openai/agents-extensions/ai-sdk` 将 AI SDK 模型适配为可由 `Agent` 运行。
- `@openai/agents-extensions/ai-sdk-ui` 将流式 Agents SDK 运行适配为可由 AI SDK UI 路由返回标准流式 `Response`。

### AI SDK 模型说明

- `@openai/agents-extensions/ai-sdk` 适配器仍处于 beta 阶段，因此建议您结合所选提供商（尤其是较小提供商）仔细测试。
- 如果您使用的是 OpenAI 模型，建议优先使用默认的 OpenAI 模型提供商，而不是此适配器。
- 受支持的 AI SDK 提供商必须暴露 `specificationVersion` 为 `v2` 或 `v3`。如果您需要旧版 v1 提供商风格，请将 [examples/ai-sdk-v1](https://github.com/openai/openai-agents-js/tree/main/examples/ai-sdk-v1) 中的模块复制到您的项目中。
- 此处不支持 Deferred Responses 工具加载流程。这包括 `toolNamespace()`、`deferLoading: true` 的函数工具，以及 `toolSearchTool()`。如果您需要工具搜索，请直接使用 OpenAI Responses 模型。请参见[工具](/openai-agents-js/zh/guides/tools#deferred-tool-loading-with-tool-search)和[模型](/openai-agents-js/zh/guides/models#responses-only-deferred-tool-loading)。

## AI SDK UI 流辅助函数

`@openai/agents-extensions/ai-sdk-ui` 提供了响应辅助函数，用于将 Agents SDK 流接入 AI SDK UI 路由：

- `createAiSdkTextStreamResponse(source, options?)` 用于纯文本流式响应。
- `createAiSdkUiMessageStreamResponse(source, options?)` 用于 `UIMessageChunk` 流式响应。

两个辅助函数都接受 `StreamedRunResult`、类流来源或兼容包装对象，并返回带有流式友好标头的 `Response`。

当您的 UI 需要工具调用或推理片段等结构化分块时，请使用 `createAiSdkUiMessageStreamResponse(...)`。当您只需要纯文本时，请使用 `createAiSdkTextStreamResponse(...)`。
