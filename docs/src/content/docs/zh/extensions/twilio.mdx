Twilio 提供了一个 [Media Streams API](https://www.twilio.com/docs/voice/media-streams)，可将电话通话中的原始音频发送到 WebSocket 服务器。此设置可用于将您的[语音智能体概述](/openai-agents-js/zh/guides/voice-agents)连接到 Twilio。您可以在 `websocket` 模式下使用默认的 Realtime Session transport，将来自 Twilio 的事件连接到您的 Realtime Session。不过，这要求您设置正确的音频格式，并自行调整中断时机，因为电话通话天然会比基于 Web 的对话引入更多延迟。

为了改善配置体验，我们创建了一个专用的传输层，用于处理与 Twilio 的连接，包括中断处理和音频转发。

<Aside type="caution">
  此适配器仍处于 beta 阶段。您可能会遇到一些边缘情况问题或 bug。 请通过 [GitHub
  issues](https://github.com/openai/openai-agents-js/issues)
  报告任何问题，我们会尽快修复。
</Aside>

## 配置

<Steps>

1. **确保您拥有一个 Twilio 账户和一个 Twilio 电话号码。**

2. **设置一个可接收来自 Twilio 事件的 WebSocket 服务器。**

   如果您在本地开发，则需要配置一个本地隧道，例如 [`ngrok`](https://ngrok.io/) 或 [Cloudflare Tunnel](https://developers.cloudflare.com/pages/how-to/preview-with-cloudflare-tunnel/)，以便 Twilio 可以访问您的本地服务器。您可以使用 `TwilioRealtimeTransportLayer` 连接到 Twilio。

3. **通过安装 extensions 包来安装 Twilio 适配器：**

   ```bash
   npm install @openai/agents-extensions
   ```

5. **将您的 `RealtimeSession` 连接到 Twilio：**

## 提示与注意事项

1. **速度至关重要。**

   为了接收来自 Twilio 的所有必要事件和音频，您应在拿到 WebSocket 连接的引用后尽快创建 `TwilioRealtimeTransportLayer` 实例，并在之后立即调用 `session.connect()`。

2. **访问原始 Twilio 事件。**

   如果您想访问 Twilio 发送的原始事件，可以监听 `RealtimeSession` 实例上的 `transport_event` 事件。来自 Twilio 的每个事件都会具有 `twilio_message` 类型，并带有一个 `message` 属性，其中包含原始事件数据。

3. **查看调试日志。**

   有时您可能会遇到问题，并希望获得更多关于当前发生情况的信息。使用 `DEBUG=openai-agents*` 环境变量将显示来自 Agents SDK 的所有调试日志。或者，您也可以仅为 Twilio 适配器启用调试日志，方法是使用 `DEBUG=openai-agents:extensions:twilio*`。
