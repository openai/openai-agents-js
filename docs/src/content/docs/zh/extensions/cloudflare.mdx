Cloudflare Workers 和其他 workerd 运行时无法使用全局 `WebSocket` 构造函数打开出站 WebSocket。为简化从这些环境连接实时智能体，extensions 包提供了一个专用传输机制，可在内部执行基于 `fetch()` 的升级。

<Aside type="caution">
  此适配器仍处于 beta 阶段。您可能会遇到一些边缘情况问题或 bug。 请通过 [GitHub
  issues](https://github.com/openai/openai-agents-js/issues)
  报告任何问题，我们会尽快修复。 对于 Workers 中的 Node.js 风格 API，可考虑启用
  `nodejs_compat`。
</Aside>

## 设置

<Steps>

1. **安装 extensions 包。**

   ```bash
   npm install @openai/agents-extensions
   ```

2. **创建传输机制并将其附加到您的会话。**

   <Code lang="typescript" code={cloudflareBasicExample} />

3. **连接您的 `RealtimeSession`。**

   ```typescript
   await session.connect({ apiKey: 'your-openai-ephemeral-or-server-key' });
   ```

</Steps>

## 说明

- Cloudflare 传输机制底层使用带有 `Upgrade: websocket` 的 `fetch()`，并跳过等待套接字 `open` 事件，这与 workerd API 保持一致。
- 使用此传输机制时，所有 `RealtimeSession` 功能（工具、护栏等）都可照常使用。
- 开发期间可使用 `DEBUG=openai-agents*` 查看详细日志。
