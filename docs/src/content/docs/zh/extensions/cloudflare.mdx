Cloudflare Workers 和其他 workerd 运行时无法使用全局 `WebSocket` 构造函数发起出站 WebSocket。为简化在这些环境中连接 实时智能体 的操作，extensions 包提供了一个专用传输，在内部通过基于 `fetch()` 的升级完成连接。

<Aside type="caution">
  此适配器仍处于测试阶段。您可能会遇到边缘情况或错误。 请通过 [GitHub
  issues](https://github.com/openai/openai-agents-js/issues)
  报告问题，我们会尽快修复。 若在 Workers 中需要 Node.js 风格的 API，考虑启用
  `nodejs_compat`。
</Aside>

## 设置

<Steps>

1. **安装扩展包。**

   ```bash
   npm install @openai/agents-extensions
   ```

2. **创建一个传输并将其附加到你的会话。**

   <Code lang="typescript" code={cloudflareBasicExample} />

3. **连接你的 `RealtimeSession`。**

   ```typescript
   await session.connect({ apiKey: 'your-openai-ephemeral-or-server-key' });
   ```

</Steps>

## 注意事项

- Cloudflare 传输在底层使用携带 `Upgrade: websocket` 的 `fetch()`，并跳过等待套接字的 `open` 事件，与 workerd 的 API 保持一致。
- 使用该传输时，所有 `RealtimeSession` 功能（工具、护栏等）都可照常工作。
- 在开发期间使用 `DEBUG=openai-agents*` 查看详细日志。
