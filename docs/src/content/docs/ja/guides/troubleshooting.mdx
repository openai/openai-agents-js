## 対応環境

OpenAI Agents SDK は次のサーバー環境でサポートされています:

- Node.js 22+
- Deno 2.35+
- Bun 1.2.5+

### 限定的なサポート

- **Cloudflare Workers**: Agents SDK は Cloudflare Workers で使用できますが、現時点ではいくつかの制限があります:
  - 現在の SDK は `nodejs_compat` を有効にする必要があります
  - トレースはリクエストの終了時に手動でフラッシュする必要があります。[トレーシング](/openai-agents-js/ja/guides/tracing#export-loop-lifecycle) を参照してください
  - Cloudflare Workers における `AsyncLocalStorage` のサポートが限定的なため、一部のトレースが正確でない可能性があります
  - 外向きの WebSocket 接続は fetch ベースのアップグレードを使用する必要があります（グローバルの `WebSocket` コンストラクターは不可）。Realtime では、`@openai/agents-extensions` の Cloudflare トランスポート（`CloudflareRealtimeTransportLayer`）を使用してください
- **Browsers**:
  - ブラウザでは現在トレーシングはサポートされていません
- **v8 isolates**:
  - 適切なブラウザ用ポリフィルを備えたバンドラを使用すれば v8 isolates 向けに SDK をバンドルできるはずですが、トレーシングは機能しません
  - v8 isolates は広範にはテストされていません

## デバッグロギング

SDK で問題が発生している場合、デバッグロギングを有効にして、何が起きているかの詳細情報を取得できます。

`DEBUG` 環境変数を `openai-agents:*` に設定してデバッグロギングを有効にします。

```bash
DEBUG=openai-agents:*
```

または、SDK の特定の部分にデバッグ範囲を絞ることもできます:

- `openai-agents:core` — SDK のメイン実行ロジック
- `openai-agents:openai` — OpenAI API 呼び出し
- `openai-agents:realtime` — リアルタイムエージェントのコンポーネント
