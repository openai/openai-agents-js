## サポート環境

OpenAI Agents SDK は次のサーバー環境でサポートされています。

- Node.js 22+
- Deno 2.35+
- Bun 1.2.5+

### 限定サポート

- **Cloudflare Workers**: Agents SDK は Cloudflare Workers で利用できますが、現時点ではいくつかの制限があります
  - SDK は現在 `nodejs_compat` を有効にする必要がある
  - リクエストの最後に手動でトレースをフラッシュする必要がある [トレーシング](/openai-agents-js/ja/guides/tracing#export-loop-lifecycle) を参照
  - Cloudflare Workers の `AsyncLocalStorage` のサポートが限定的なため、一部のトレースが正確でない可能性がある
  - 外向きの WebSocket 接続は fetch ベースのアップグレードを使用する必要がある（グローバルの `WebSocket` コンストラクターは不可）。Realtime の場合は `@openai/agents-extensions` の Cloudflare トランスポート（`CloudflareRealtimeTransportLayer`）を使用する
- **ブラウザ**:
  - ブラウザでは現在トレーシングはサポートされていない
- **v8 アイソレート**:
  - 適切なブラウザポリフィルを備えたバンドラーを使用すれば v8 アイソレート向けに SDK をバンドルできるはずだが、トレーシングは動作しない
  - v8 アイソレートは広範にはテストされていない

## デバッグログ

SDK の利用中に問題が発生した場合、デバッグログを有効にして状況の詳細を取得できます。

`DEBUG` 環境変数を `openai-agents:*` に設定してデバッグログを有効化します。

```bash
DEBUG=openai-agents:*
```

または、SDK の特定の部分にスコープを限定してデバッグを有効にできます。

- `openai-agents:core` — SDK の主要な実行ロジック
- `openai-agents:openai` — OpenAI API 呼び出し
- `openai-agents:realtime` — リアルタイムエージェントのコンポーネント
