現在の主なユースケースは、機微なツール実行に対して承認を求めることです。

### フロー

1. エージェントがツール（複数含む）を呼び出すと判断した場合、`needsApproval` を評価してそのツールに承認が必要かを確認します
2. 承認が必要な場合、エージェントは承認がすでに許可または拒否されているかを確認します
   - 承認が許可または拒否されていない場合、ツールはツール呼び出しを実行できないことを示す静的メッセージをエージェントに返します
   - 承認 / 拒否が未処理の場合、ツール承認リクエストがトリガーされます
3. エージェントはすべてのツール承認リクエストを収集し、実行を中断します
4. 中断がある場合、[エージェントの実行結果](/openai-agents-js/ja/guides/results) に、保留中の手順を記述した `interruptions` 配列が含まれます。ツール呼び出しに確認が必要なときは、`type: "tool_approval_item"` の `ToolApprovalItem` が表示されます
5. ツール呼び出しを許可または拒否するには、`result.state.approve(interruption)` または `result.state.reject(interruption)` を呼び出します
6. すべての中断に対応したら、`result.state` を `runner.run(agent, state)` に渡して実行を再開できます。ここで `agent` は全体の実行をトリガーした元のエージェントです
7. フローは手順 1 から再開されます

## 長い承認所要時間への対応

Human in the loop (人間の介入) フローは、サーバーを稼働し続けなくても、長時間中断できるように設計されています。要求をいったん停止し、後で続行する必要がある場合は、状態をシリアライズして後で再開できます。

`JSON.stringify(result.state)` を使って状態をシリアライズし、後で `RunState.fromString(agent, serializedState)` にシリアライズ済み状態を渡すことで再開できます。ここで `agent` は全体の実行をトリガーしたエージェントのインスタンスです。

これにより、シリアライズした状態をデータベースやリクエストと一緒に保存できます。

### 保留タスクのバージョニング

<Aside>
  これは主に、エージェントに変更を加えながら、シリアライズした状態を
  長期間保存しようとしている場合に該当します。
</Aside>

承認リクエストに時間がかかり、エージェント定義を意味のある形でバージョン管理したい、または Agents SDK のバージョンを上げたい場合は、パッケージエイリアスを使って Agents SDK の 2 つのバージョンを並行インストールし、独自の分岐ロジックを実装することを現在は推奨します。

実務上は、自身のコードにバージョン番号を割り当て、それをシリアライズした状態と一緒に保存し、デシリアライズを正しいコードのバージョンへ誘導することを意味します。
