Cloudflare Workers やその他の workerd ランタイムでは、グローバルの `WebSocket` コンストラクタを使って外向きの WebSocket を開くことはできません。これらの環境から リアルタイムエージェント を簡単に接続できるように、extensions パッケージは `fetch()` ベースのアップグレードを内部で行う専用のトランスポートを提供します。

<Aside type="caution">
  このアダプターはまだベータ版です。レアケースの問題やバグに遭遇する可能性があります。
  問題があれば [GitHub
  issues](https://github.com/openai/openai-agents-js/issues)
  からご報告ください。迅速に修正します。Workers で Node.js 風の API
  を使う場合は、`nodejs_compat` の有効化をご検討ください。
</Aside>

## セットアップ

<Steps>

1. **extensions パッケージをインストールします。**

   ```bash
   npm install @openai/agents-extensions
   ```

2. **トランスポートを作成し、セッションに関連付けます。**

   <Code lang="typescript" code={cloudflareBasicExample} />

3. **`RealtimeSession` に接続します。**

   ```typescript
   await session.connect({ apiKey: 'your-openai-ephemeral-or-server-key' });
   ```

</Steps>

## 注意事項

- Cloudflare 用トランスポートは内部的に `fetch()` と `Upgrade: websocket` を使用し、workerd の API に合わせてソケットの `open` イベントを待ちません
- このトランスポート使用時でも、すべての `RealtimeSession` 機能（ツール、ガードレールなど）は通常どおり動作します
- 開発中の詳細ログを確認するには `DEBUG=openai-agents*` を使用します
