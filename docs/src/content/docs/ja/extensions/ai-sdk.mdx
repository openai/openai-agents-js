Agents SDK は、標準で Responses API または Chat Completions API を通じて OpenAI モデルと連携できます。ただし、別のモデルを使いたい場合は、[Vercel AI SDK](https://sdk.vercel.ai/) が対応するさまざまなモデルを、このアダプター経由で Agents SDK に組み込めます。

## 適切な連携方法の選択

`@openai/agents-extensions` には、関連する 2 つの連携があります。

- `@openai/agents-extensions/ai-sdk` は AI SDK モデルを適合させ、`Agent` がそのモデル上で実行できるようにします
- `@openai/agents-extensions/ai-sdk-ui` はストリーミングされた Agents SDK 実行を適合させ、AI SDK UI ルートが標準的なストリーミング `Response` を返せるようにします

### AI SDK モデルに関する注意点

- `@openai/agents-extensions/ai-sdk` アダプターはまだベータ版のため、特に小規模プロバイダーを使う場合は慎重な検証をおすすめします
- OpenAI モデルを使う場合は、このアダプターではなくデフォルトの OpenAI モデルプロバイダーを優先してください
- 対応する AI SDK プロバイダーは `specificationVersion` が `v2` または `v3` である必要があります。古い v1 プロバイダースタイルが必要な場合は、[examples/ai-sdk-v1](https://github.com/openai/openai-agents-js/tree/main/examples/ai-sdk-v1) のモジュールをプロジェクトにコピーしてください
- ここでは Deferred Responses のツール読み込みフローはサポートされません。これには `toolNamespace()`、`deferLoading: true` を持つ関数ツール、`toolSearchTool()` が含まれます。ツール検索が必要な場合は、OpenAI Responses モデルを直接使用してください。[ツール](/openai-agents-js/ja/guides/tools#deferred-tool-loading-with-tool-search) と [モデル](/openai-agents-js/ja/guides/models#responses-only-deferred-tool-loading) を参照してください

## AI SDK UI ストリームヘルパー

`@openai/agents-extensions/ai-sdk-ui` は、Agents SDK ストリームを AI SDK UI ルートに接続するための応答ヘルパーを提供します。

- `createAiSdkTextStreamResponse(source, options?)`: プレーンテキストのストリーミング応答用
- `createAiSdkUiMessageStreamResponse(source, options?)`: `UIMessageChunk` のストリーミング応答用

どちらのヘルパーも `StreamedRunResult`、ストリームライクなソース、または互換ラッパーオブジェクトを受け取り、ストリーミングに適したヘッダー付きの `Response` を返します。

ツール呼び出しや推論パートなどの構造化チャンクが UI で必要な場合は `createAiSdkUiMessageStreamResponse(...)` を使います。プレーンテキストだけが必要な場合は `createAiSdkTextStreamResponse(...)` を使います。
