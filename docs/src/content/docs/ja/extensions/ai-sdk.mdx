## リトライ

`modelSettings.retry` は AI SDK ベースのモデルでも動作します。これは、リトライがデフォルトの OpenAI プロバイダーだけでなく Agents ランタイム側で実装されているためです。

つまり、他の場所で使うのと同じリトライ設定を適用できます:

- `Agent`、`Runner`、またはその両方に `modelSettings.retry` を設定する
- `networkError()`、`httpStatus([...])`、`providerSuggested()` などの `retryPolicies` を組み合わせる
- `providerSuggested()` が有効なのは、ラップされた AI SDK モデルがアダプターを通じてリトライの助言を表出できる場合だけであることに注意する

`aisdk(openai(...))` を使った完全な例については、[`examples/ai-sdk/retry.ts`](https://github.com/openai/openai-agents-js/tree/main/examples/ai-sdk/retry.ts) を参照してください。ストリーミングや状態を持つ後続リクエストに対する安全境界を含むリトライ API 自体については、[モデルガイド](/openai-agents-js/ja/guides/models#model-retries) を参照してください。

## 適切な連携方法の選択

`@openai/agents-extensions` には、関連する 2 つの連携があります:

- `@openai/agents-extensions/ai-sdk` は AI SDK モデルを適応させ、`Agent` がその上で実行できるようにします
- `@openai/agents-extensions/ai-sdk-ui` はストリーミングされた Agents SDK 実行を適応させ、AI SDK UI ルートが標準的なストリーミング `Response` を返せるようにします

### AI SDK モデルに関する注意

- `@openai/agents-extensions/ai-sdk` アダプターはまだ beta なので、特に小規模なプロバイダーでは、選択したプロバイダーで慎重にテストすることをおすすめします
- OpenAI モデルを使用している場合は、このアダプターではなくデフォルトの OpenAI モデルプロバイダーを優先してください
- 対応する AI SDK プロバイダーは、`specificationVersion` として `v2` または `v3` を公開している必要があります。古い v1 プロバイダースタイルが必要な場合は、[examples/ai-sdk-v1](https://github.com/openai/openai-agents-js/tree/main/examples/ai-sdk-v1) のモジュールをプロジェクトにコピーしてください
- コンピュータツールをこのアダプター経由で使う場合は、display メタデータが必要です。ツールに `environment` と `dimensions` の両方のメタデータが含まれていることを確認してください
- Deferred Responses のツール読み込みフローはここではサポートされていません。これには `toolNamespace()`、`deferLoading: true` を持つ関数ツール、`toolSearchTool()` が含まれます。ツール検索が必要な場合は、OpenAI Responses モデルを直接使ってください。[ツールガイド](/openai-agents-js/ja/guides/tools#deferred-tool-loading-with-tool-search) と [モデルガイド](/openai-agents-js/ja/guides/models#responses-only-deferred-tool-loading) を参照してください

## AI SDK UI ストリームヘルパー

`@openai/agents-extensions/ai-sdk-ui` は、Agents SDK のストリームを AI SDK UI ルートに接続するためのレスポンスヘルパーを提供します:

- プレーンテキストのストリーミング応答には `createAiSdkTextStreamResponse(source, options?)`
- `UIMessageChunk` のストリーミング応答には `createAiSdkUiMessageStreamResponse(source, options?)`

どちらのヘルパーも `StreamedRunResult`、ストリーム風のソース、または互換ラッパーオブジェクトを受け取り、ストリーミング向けヘッダー付きの `Response` を返します。

ツール呼び出しや reasoning パートのような構造化チャンクが UI に必要な場合は、`createAiSdkUiMessageStreamResponse(...)` を使ってください。プレーンテキストだけが必要な場合は、`createAiSdkTextStreamResponse(...)` を使ってください。

どちらのヘルパーも、`options` を通じて任意のレスポンス設定を受け付けます:

- `headers`: ストリーミング応答にマージする追加のレスポンスヘッダー
- `status`: 返される `Response` の HTTP ステータスコード
- `statusText`: 返される `Response` の HTTP ステータステキスト
