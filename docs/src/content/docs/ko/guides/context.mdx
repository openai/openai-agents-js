컨텍스트는 여러 의미로 쓰이는 용어입니다. 여기서 중요할 수 있는 컨텍스트는 크게 두 가지입니다:

1. 실행 중 코드가 접근할 수 있는 **로컬 컨텍스트**: 도구에 필요한 의존성이나 데이터, `onHandoff` 같은 콜백, 라이프사이클 훅
2. 언어 모델이 응답을 생성할 때 볼 수 있는 **에이전트/LLM 컨텍스트**

## 로컬 컨텍스트

로컬 컨텍스트는 `RunContext<T>` 타입으로 표현됩니다. 상태나 의존성을 담을 객체를 직접 만들고, 이를 `Runner.run()`에 전달합니다. 모든 도구 호출과 훅은 `RunContext` 래퍼를 받아 그 객체를 읽거나 수정할 수 있습니다.

하나의 실행에 참여하는 모든 에이전트, 도구, 훅은 동일한 컨텍스트 **타입**을 사용해야 합니다.

로컬 컨텍스트는 다음과 같은 용도로 사용합니다:

- 실행에 관한 데이터(사용자 이름, ID 등)
- 로거나 데이터 fetcher 같은 의존성
- 헬퍼 함수

<Aside type="note">
  컨텍스트 객체는 LLM으로 전송되지 **않습니다**. 완전히 로컬에서만 사용되며
  자유롭게 읽고 쓸 수 있습니다.
</Aside>

단일 실행 내에서 파생된 컨텍스트는 동일한 기반 앱 컨텍스트, 승인, 사용량 추적을 공유합니다. 중첩된 `agent.asTool()` 실행은 다른 `toolInput`을 연결할 수 있지만, 기본적으로 앱 상태의 격리된 복사본을 받지는 않습니다.

### `RunContext`에서 제공하는 항목

`RunContext<T>`는 앱에서 정의한 컨텍스트 객체를 감싸는 래퍼입니다. 실제로는 보통 다음 항목들을 가장 많이 사용합니다:

- `runContext.context`: 앱 고유의 가변 상태와 의존성
- `runContext.usage`: 현재 실행의 집계된 토큰/요청 사용량
- `runContext.toolInput`: 현재 실행이 `agent.asTool()` 내부에서 수행될 때의 structured outputs 입력
- `runContext.approveTool(...)` / `runContext.rejectTool(...)`: 승인 상태를 프로그래밍 방식으로 업데이트해야 할 때 사용

앱에서 정의한 객체는 `runContext.context`뿐입니다. 나머지 필드는 SDK가 관리하는 런타임 메타데이터입니다.

나중에 [휴먼 인 더 루프 (HITL)](/openai-agents-js/ko/guides/human-in-the-loop)를 위해 [`RunState`](/openai-agents-js/guides/results#state)를 직렬화하면, 그 런타임 메타데이터도 상태와 함께 저장됩니다. 직렬화된 상태를 영속화하거나 전송할 계획이라면 `runContext.context`에 비밀 정보를 넣지 않는 것이 좋습니다.

<Aside type="note">
  `RunContext`는 현재 실행을 위한 로컬 앱 상태를 다룹니다. 대화 상태는 별개의
  관심사입니다. 턴을 이어가는 방식에 따라 `result.history`, `session`,
  `conversationId`, 또는 `previousResponseId`를 사용하세요. 이 판단 기준은
  [에이전트
  실행](/openai-agents-js/guides/running-agents#state-and-conversation-management)에서
  확인할 수 있습니다.
</Aside>

`RunContext`를 서브클래싱하는 경우, 중첩 실행이나 파생 실행에서도 의존하는 서브클래스 고유의 인스턴스 상태가 유지되는지 확인하세요. SDK는 중첩 실행 중 내부적으로 fork된 컨텍스트를 생성합니다.

## 에이전트/LLM 컨텍스트

LLM이 호출될 때, 모델이 볼 수 있는 데이터는 대화 기록에서 오는 것뿐입니다. 추가 정보를 사용할 수 있게 하려면 몇 가지 방법이 있습니다:

1. 에이전트 `instructions`에 추가합니다. 이는 system 또는 developer message라고도 합니다. 정적 문자열일 수도 있고, 컨텍스트를 받아 문자열을 반환하는 함수일 수도 있습니다.
2. `Runner.run()`을 호출할 때 `input`에 포함합니다. 이는 instructions 기법과 비슷하지만, 메시지를 [chain of command](https://cdn.openai.com/spec/model-spec-2024-05-08.html#follow-the-chain-of-command)에서 더 낮은 위치에 둘 수 있습니다.
3. 함수 도구를 통해 노출하여 LLM이 필요할 때 데이터를 가져오도록 합니다.
4. retrieval 또는 웹 검색 도구를 사용해 파일, 데이터베이스, 웹의 관련 데이터에 응답을 기반하도록 합니다.
