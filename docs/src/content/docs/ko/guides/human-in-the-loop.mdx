이 가이드는 SDK에 내장된 휴먼인더루프 (HITL) 지원을 사용하여 사람의 개입을 기준으로 에이전트 실행을 일시 중지하고 다시 재개하는 방법을 보여줍니다.

현재 이 기능의 주요 사용 사례는 민감한 도구 실행에 대한 승인을 요청하는 것입니다.

### 흐름

1. 에이전트가 하나 이상의 도구를 호출하기로 결정하면 `needsApproval`을 평가하여 이 도구에 승인이 필요한지 확인함
2. 승인이 필요한 경우 에이전트는 승인이 이미 허용되었거나 거부되었는지 확인함
   - 승인이 허용되었거나 거부되지 않은 경우, 도구 호출을 실행할 수 없다는 정적 메시지를 에이전트에 반환함
   - 승인/거부가 없는 경우 도구 승인 요청을 트리거함
3. 에이전트는 모든 도구 승인 요청을 모으고 실행을 인터럽션(중단 처리)함
4. 인터럽션(중단 처리)이 있으면 [Results](/openai-agents-js/ko/guides/result)에 보류 중인 단계를 설명하는 `interruptions` 배열이 포함됨. 도구 호출에 확인이 필요한 경우 `type: "tool_approval_item"`을 가진 `ToolApprovalItem`이 나타남
5. `result.state.approve(interruption)` 또는 `result.state.reject(interruption)`를 호출하여 도구 호출을 승인하거나 거부할 수 있음
6. 모든 인터럽션(중단 처리)을 처리한 후, `agent`가 전체 실행을 트리거한 원래 에이전트일 때 `runner.run(agent, state)`에 `result.state`를 다시 전달하여 실행을 재개할 수 있음
7. 흐름은 1단계부터 다시 시작됨

## 긴 승인 대기 시간 처리

휴먼인더루프 (HITL) 흐름은 서버를 계속 실행하지 않고도 오랜 시간 인터럽션(중단 처리)이 가능하도록 설계되었습니다. 요청을 종료하고 나중에 계속해야 한다면 상태를 직렬화하여 이후에 재개할 수 있습니다.

`JSON.stringify(result.state)`로 상태를 직렬화하고, 직렬화된 상태를 `RunState.fromString(agent, serializedState)`에 전달하여 나중에 재개할 수 있습니다. 여기서 `agent`는 전체 실행을 트리거한 에이전트 인스턴스입니다.

이렇게 하면 직렬화된 상태를 데이터베이스나 요청과 함께 저장할 수 있습니다.

### 보류 중 작업의 버전 관리

<Aside>
  이는 주로 직렬화된 상태를 오래 저장해야 하면서 에이전트를 변경하려는 경우에
  적용됩니다.
</Aside>

승인 요청에 시간이 오래 걸리고 에이전트 정의를 의미 있게 버전 관리하거나 Agents SDK 버전을 올릴 계획이라면, 현재로서는 패키지 별칭을 사용해 두 버전의 Agents SDK를 병렬로 설치하고 자체 분기 로직을 구현할 것을 권장합니다.

실제로는 코드 자체에 버전 번호를 부여하고 이를 직렬화된 상태와 함께 저장한 다음, 역직렬화를 코드의 올바른 버전으로 유도하는 방식을 의미합니다.
