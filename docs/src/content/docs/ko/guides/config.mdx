WebSocket 전송 방식을 활성화하려면 `setOpenAIResponsesTransport('websocket')`을 사용하고, 다시 전환하려면 `setOpenAIResponsesTransport('http')`을 사용하세요. 프록시나 게이트웨이를 통해 WebSocket 트래픽을 라우팅하는 경우 `OPENAI_WEBSOCKET_BASE_URL`을 설정하거나 `OpenAIProvider`에서 `websocketBaseURL`을 구성하세요.

### 디버그 로깅

SDK는 디버그 로깅에 [`debug`](https://www.npmjs.com/package/debug) 패키지를 사용합니다. 상세 로그를 보려면 `DEBUG` 환경 변수를 `openai-agents*`로 설정하세요.

```bash
export DEBUG=openai-agents*
```

세션 지속성 작업을 로깅하려면 `OPENAI_AGENTS__DEBUG_SAVE_SESSION=1`을 설정하세요.

`@openai/agents`의 `getLogger(namespace)`를 사용하여 자체 모듈에 사용할 네임스페이스 로거를 가져올 수 있습니다.

#### 로그의 민감한 데이터

일부 로그에는 사용자 데이터가 포함될 수 있습니다. 다음 환경 변수를 설정하여 이러한 로그를 비활성화하세요.

LLM 입력 및 출력 로깅을 비활성화하려면 다음과 같이 설정합니다.

```bash
export OPENAI_AGENTS_DONT_LOG_MODEL_DATA=1
```

이 설정은 SDK 오류 로그에서 모델 관련 오류 객체와 세부 정보도 수정하여 안전한 오류 유형만 남깁니다.

도구 입력 및 출력 로깅을 비활성화하려면 다음과 같이 설정합니다.

```bash
export OPENAI_AGENTS_DONT_LOG_TOOL_DATA=1
```

이 설정은 SDK 오류 로그에서 도구 실행, 승인 형식 처리, Realtime 도구 처리 실패를 비롯한 도구 관련 오류 객체와 세부 정보도 수정합니다.
