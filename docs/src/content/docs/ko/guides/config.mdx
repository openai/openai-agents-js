이 페이지에서는 기본 OpenAI 클라이언트, 전송 방식, 트레이싱 내보내기 키, 디버그 로깅 동작처럼 일반적으로 앱 시작 시 한 번 설정하는 **SDK 전역 기본값**을 다룹니다. 이러한 설정은 기본적으로 프로세스 전체에 적용되므로, 에이전트별 또는 실행별 조정이 아닌 공통 설정에 적합합니다.

WebSocket 전송 방식을 활성화하려면 `setOpenAIResponsesTransport('websocket')`을 사용하고, 다시 전환하려면 `setOpenAIResponsesTransport('http')`를 사용하세요. WebSocket 트래픽을 프록시나 게이트웨이를 통해 라우팅하는 경우 `OPENAI_WEBSOCKET_BASE_URL`을 설정하거나 `OpenAIProvider`에서 `websocketBaseURL`을 구성하세요.

### 디버그 로깅

SDK는 디버그 로깅에 [`debug`](https://www.npmjs.com/package/debug) 패키지를 사용합니다. 상세 로그를 확인하려면 `DEBUG` 환경 변수를 `openai-agents*`로 설정하세요.

```bash
export DEBUG=openai-agents*
```

세션 지속성 활동을 기록하려면 `OPENAI_AGENTS__DEBUG_SAVE_SESSION=1`을 설정하세요.

`@openai/agents`의 `getLogger(namespace)`를 사용하여 자체 모듈용 네임스페이스 로거를 가져올 수 있습니다.

#### 로그의 민감한 데이터

관련 오류 객체와 세부 정보를 포함한 모델 및 도구 데이터는 기본적으로 로그에 포함되지 않습니다. 디버깅 중 이러한 세부 정보가 필요한 경우 프로세스에서 민감한 데이터 로깅을 명시적으로 활성화할 수 있습니다.

<Code
  lang="typescript"
  code={setSensitiveDataLoggingEnabledExample}
  title="민감한 데이터 로깅 활성화"
/>

로그가 안전하게 처리되는 환경에서만 이 기능을 활성화하세요. 프로그래밍 방식의 설정은 모델 및 도구 데이터를 모두 제어하며 아래 환경 변수보다 우선합니다. 민감한 데이터 로깅을 다시 비활성화하려면 `setSensitiveDataLoggingEnabled(false)`를 호출하세요.

환경 변수를 사용하여 모델 및 도구 데이터 로깅을 개별적으로 활성화할 수도 있습니다. 해당 `OPENAI_AGENTS_DONT_LOG_*` 변수를 `0` 또는 `false`로 설정하세요.

```bash
export OPENAI_AGENTS_DONT_LOG_MODEL_DATA=0
export OPENAI_AGENTS_DONT_LOG_TOOL_DATA=0
```

두 변수 중 하나를 `1` 또는 `true`로 설정하면 해당 데이터와 관련 오류 세부 정보가 계속 마스킹됩니다. 설정되지 않았거나 인식할 수 없는 값에도 안전한 기본값이 적용되어 민감한 데이터가 로그에 포함되지 않습니다.
