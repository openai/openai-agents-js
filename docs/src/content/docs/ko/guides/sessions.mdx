세션은 Agents SDK에 **지속형 메모리 계층**을 제공합니다. `Session` 인터페이스를 구현하는 어떤 객체든 `Runner.run`에 제공하면, 나머지는 SDK가 처리합니다. 세션이 있으면 러너는 자동으로 다음을 수행합니다:

1. 이전에 저장된 대화 항목을 가져와 다음 턴 앞에 붙임
2. 각 실행이 완료된 후 새로운 user 입력과 assistant 출력 영속화
3. 새로운 user 텍스트로 러너를 호출하든 인터럽트된 `RunState`에서 재개하든, 이후 턴을 위해 세션을 유지

이로써 `toInputList()`를 수동으로 호출하거나 턴 간 기록을 이어 붙일 필요가 없어집니다. TypeScript SDK는 두 가지 구현을 제공합니다: Conversations API용 `OpenAIConversationsSession`과 로컬 개발용 `MemorySession`. 이들은 `Session` 인터페이스를 공유하므로, 원하는 스토리지 백엔드를 연결할 수 있습니다. Conversations API 외의 영감이 필요하다면 `examples/memory/` 아래의 샘플 세션 백엔드(Prisma, 파일 기반 등)를 살펴보세요. OpenAI Responses 모델을 사용할 때는 어떤 세션이든 `OpenAIResponsesCompactionSession`으로 감싸 [`responses.compact`](https://platform.openai.com/docs/api-reference/responses/compact)를 통해 저장된 대화를 자동 축소하세요.

> Tip: 이 페이지의 `OpenAIConversationsSession` 예제를 실행하려면 `OPENAI_API_KEY` 환경 변수를 설정하거나(또는 세션 생성 시 `apiKey` 제공) SDK가 Conversations API를 호출할 수 있도록 하세요.

## 러너가 세션을 사용하는 방식

- 각 실행 전에 세션 히스토리를 가져와 새 턴의 입력과 병합하고, 결합된 목록을 에이전트에 전달함
- 비스트리밍 실행 후 한 번의 `session.addItems()` 호출로 최신 턴의 원래 user 입력과 모델 출력을 모두 영속화함
- 스트리밍 실행의 경우 먼저 user 입력을 기록하고, 턴이 완료되면 스트리밍된 출력을 덧붙임
- `RunResult.state`에서 재개할 때(승인 또는 기타 인터럽션) 동일한 `session`을 계속 전달함. 재개된 턴은 입력을 다시 준비하지 않고 메모리에 추가됨

## OpenAI Responses 히스토리 자동 압축

`OpenAIResponsesCompactionSession`은 어떤 `Session`이든 데코레이트하며, 대화 축약을 위해 OpenAI Responses API에 의존합니다. 각 턴을 영속화한 후 러너는 최신 `responseId`를 `runCompaction`에 전달하고, 결정 훅이 true를 반환하면 `responses.compact`를 호출합니다. 기본 트리거는 user가 아닌 항목이 최소 10개 누적되면 압축하며, 토큰 수나 사용자 정의 휴리스틱에 기반해 결정을 내리려면 `shouldTriggerCompaction`을 재정의하세요. 이 데코레이터는 압축된 출력으로 기반 세션을 지우고 다시 작성하므로, 서버가 관리하는 다른 히스토리 플로우를 사용하는 `OpenAIConversationsSession`과는 함께 사용하지 마세요.

## 승인 및 재개 가능 실행 처리

Human-in-the-loop 플로우는 종종 승인을 기다리기 위해 실행을 일시 중지합니다:

```typescript
const result = await runner.run(agent, 'Search the itinerary', {
  session,
  stream: true,
});

if (result.requiresApproval) {
  // ... collect user feedback, then resume the agent in a later turn
  const continuation = await runner.run(agent, result.state, { session });
  console.log(continuation.finalOutput);
}
```

이전 `RunState`에서 재개할 때, 새 턴은 단일 대화 히스토리를 보존하기 위해 동일한 메모리 레코드에 추가됩니다. Human-in-the-loop (HITL) 플로우는 완전히 호환됩니다. 승인 체크포인트는 여전히 `RunState`를 통해 왕복되고, 세션은 전체 대화록을 유지합니다.
