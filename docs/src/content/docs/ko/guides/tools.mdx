도구를 사용하면 에이전트가 데이터를 가져오고, 외부 API를 호출하고, 코드를 실행하거나, 심지어 컴퓨터를 사용할 수 있도록 **동작을 수행**합니다. JavaScript/TypeScript SDK는 여섯 가지 카테고리를 지원합니다:

1. **Hosted OpenAI tools** – OpenAI 서버에서 모델과 함께 실행됩니다. _(웹 검색, 파일 검색, code interpreter, 이미지 생성)_
2. **Local built-in tools** – 사용자의 환경에서 실행됩니다. _(컴퓨터 사용, shell, apply_patch)_
3. **Function tools** – 로컬 함수를 JSON 스키마로 래핑하여 LLM이 호출할 수 있게 합니다.
4. **Agents as tools** – 전체 에이전트를 호출 가능한 도구로 노출합니다.
5. **MCP servers** – Model context protocol 서버(로컬 또는 원격)를 연결합니다.
6. **Experimental: Codex tool** – Codex SDK를 함수 도구로 래핑하여 워크스페이스 인식 작업을 실행합니다.

## 2. 로컬 기본 제공 도구

로컬 기본 제공 도구는 사용자의 환경에서 실행되며 구현을 제공해야 합니다:

- **컴퓨터 사용** – `Computer` 인터페이스를 구현하고 `computerTool()`에 전달하세요.
- **Shell** – `Shell` 인터페이스를 구현하고 `shellTool()`에 전달하세요.
- **패치 적용** – `Editor` 인터페이스를 구현하고 `applyPatchTool()`에 전달하세요.

이 도구들은 로컬에서 실행되며 OpenAI에서 호스팅되지 **않습니다**. 런타임에서 파일, 터미널 또는 GUI 자동화에 직접 접근해야 할 때 사용하세요. 도구 호출은 여전히 OpenAI 모델의 응답에 의해 요청되지만, 애플리케이션이 이를 로컬에서 실행해야 합니다.

### 옵션 레퍼런스

| Field              | Required | Description                                                                                                                                                          |
| ------------------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`             | No       | 기본값은 함수 이름입니다(예: `get_weather`).                                                                                                                         |
| `description`      | Yes      | LLM에 표시되는 명확하고 사람이 읽기 쉬운 설명                                                                                                                        |
| `parameters`       | Yes      | Zod 스키마 또는 raw JSON 스키마 객체. Zod 매개변수를 사용하면 자동으로 **strict** 모드가 활성화됩니다.                                                               |
| `strict`           | No       | `true`(기본값)일 때, 인수가 유효성 검사를 통과하지 못하면 SDK가 모델 오류를 반환합니다. 퍼지 매칭이 필요하면 `false`로 설정하세요.                                   |
| `execute`          | Yes      | `(args, context) => string \| unknown \| Promise<...>` – 비즈니스 로직. 문자열이 아닌 출력은 모델을 위해 직렬화됩니다. 선택적 두 번째 매개변수는 `RunContext`입니다. |
| `errorFunction`    | No       | 내부 오류를 사용자에게 표시할 문자열로 변환하는 커스텀 핸들러 `(context, error) => string`                                                                           |
| `needsApproval`    | No       | 실행 전에 사람의 승인이 필요합니다. [휴먼 인 더 루프 (HITL)](/openai-agents-js/ko/guides/human-in-the-loop) 가이드를 참조하세요.                                     |
| `isEnabled`        | No       | 실행별로 조건부 노출; 불리언 또는 프레디케이트를 허용합니다                                                                                                          |
| `inputGuardrails`  | No       | 도구 실행 전에 실행되는 가드레일로, 거부하거나 예외를 던질 수 있습니다. [가드레일](/openai-agents-js/ko/guides/guardrails#tool-guardrails)을 참조하세요.             |
| `outputGuardrails` | No       | 도구 실행 후에 실행되는 가드레일로, 거부하거나 예외를 던질 수 있습니다. [가드레일](/openai-agents-js/ko/guides/guardrails#tool-guardrails)을 참조하세요.             |

## 4. Agents as tools

대화를 완전히 핸드오프하지 않고 한 에이전트가 다른 에이전트를 *보조*하도록 하고 싶을 수 있습니다. `agent.asTool()`을 사용하세요:

<Code lang="typescript" code={agentsAsToolsExample} title="Agents as tools" />

내부적으로 SDK는 다음을 수행합니다:

- 단일 `input` 매개변수를 갖는 함수 도구를 생성
- 도구가 호출되면 해당 입력으로 하위 에이전트를 실행
- 마지막 메시지 또는 `customOutputExtractor`로 추출된 출력을 반환

에이전트를 도구로 실행하면, Agents SDK는 기본 설정으로 러너를 생성하고 함수 실행 내에서 해당 에이전트를 실행합니다. `runConfig` 또는 `runOptions`의 어떤 속성이든 제공하려면, 러너 동작을 사용자 지정하려 `asTool()` 메서드에 전달할 수 있습니다.

또한 `needsApproval`과 `isEnabled`를 `asTool()` 옵션에서 에이전트 도구에 설정해 휴먼 인 더 루프 흐름 및 조건부 도구 제공과 통합할 수 있습니다.

### 에이전트 도구의 스트리밍 이벤트

에이전트 도구는 모든 중첩 실행 이벤트를 앱으로 스트리밍할 수 있습니다. 도구를 구성하는 방식에 맞는 훅 스타일을 선택하세요:

<Code
  lang="typescript"
  code={agentsAsToolsStreamingExample}
  title="스트리밍 에이전트 도구"
/>

- 이벤트 타입은 `RunStreamEvent['type']`과 일치합니다: `raw_model_stream_event`, `run_item_stream_event`, `agent_updated_stream_event`
- `onStream`은 가장 단순한 전체 캡처 방식으로, 도구를 인라인으로 선언할 때 잘 동작합니다(`tools: [agent.asTool({ onStream })]`). 이벤트별 라우팅이 필요 없다면 사용하세요.
- `on(eventName, handler)`는 선택적으로(또는 `'*'`로) 구독할 수 있게 해 주며, 더 세밀한 처리가 필요하거나 생성 후 리스너를 부착하고자 할 때 적합합니다.
- `onStream` 또는 어떤 `on(...)` 핸들러든 제공하면 agent-as-tool은 자동으로 스트리밍 모드로 동작합니다. 제공하지 않으면 논스트리밍 경로로 유지됩니다.
- 핸들러는 병렬로 호출되므로 느린 `onStream` 콜백이 `on(...)` 핸들러를 차단하지 않습니다(그 반대도 마찬가지).
- `toolCallId`는 도구가 모델의 도구 호출을 통해 호출된 경우 제공됩니다. 직접 `invoke()` 호출 또는 공급자 특성에 따라 생략될 수 있습니다.

## 6. 실험적: Codex 도구

`@openai/agents-extensions/experimental/codex`는 `codexTool()`을 제공합니다. 이 함수 도구는 모델의 도구 호출을 Codex SDK로 라우팅하여 에이전트가 워크스페이스 범위 작업(shell, 파일 편집, MCP 도구)을 자율적으로 실행할 수 있게 합니다. 이 표면은 실험적이며 변경될 수 있습니다.

빠른 시작:

<Code lang="typescript" code={codexToolExample} title="실험적 Codex 도구" />

알아둘 점:

- 인증: `CODEX_API_KEY`(권장) 또는 `OPENAI_API_KEY`를 제공하거나 `codexOptions.apiKey`를 전달하세요
- 입력: 엄격한 스키마 — `inputs`에는 최소 하나의 `{ type: 'text', text }` 또는 `{ type: 'local_image', path }`가 포함되어야 합니다
- 안전: `sandboxMode`를 `workingDirectory`와 함께 사용하세요. 디렉터리가 Git 저장소가 아니라면 `skipGitRepoCheck`를 설정하세요
- 동작: `persistSession: true`는 단일 Codex 스레드를 재사용하고 해당 `threadId`를 반환합니다. 재개 가능한 작업을 위해 이를 노출할 수 있습니다
- 스트리밍: `onStream`은 Codex 이벤트(추론, 명령 실행, MCP 도구 호출, 파일 변경, 웹 검색)를 미러링하므로 진행 상황을 로깅하거나 트레이싱할 수 있습니다
- 출력: 도구 결과에는 `response`, `usage`, `threadId`가 포함되며 Codex 토큰 사용량은 `RunContext`에 기록됩니다
- 구조: 형식화된 출력이 필요할 때 `outputSchema`는 각 턴의 Codex 응답에 structured outputs을 강제합니다

## 모범 사례

- **짧고 명확한 설명** – 도구가 _무엇을_ 하고 _언제 사용하는지_ 설명하세요
- **입력 검증** – 가능하면 Zod 스키마로 엄격한 JSON 검증을 사용하세요
- **에러 핸들러에서 부작용 회피** – `errorFunction`은 예외를 던지지 말고 유용한 문자열을 반환해야 합니다
- **도구당 하나의 책임** – 작고 조합 가능한 도구는 더 나은 모델 추론으로 이어집니다
