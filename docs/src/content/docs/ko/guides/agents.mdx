에이전트는 OpenAI Agents SDK의 주요 기본 구성 요소입니다. **에이전트**는 다음과 같이 구성된 대규모 언어 모델(LLM)입니다:

- **Instructions** – 모델에게 _자신이 누구인지_ 와 _어떻게 응답해야 하는지_ 알려주는 시스템 프롬프트
- **Model** – 호출할 OpenAI 모델과 선택적 모델 튜닝 매개변수
- **Tools** – 작업을 수행하기 위해 LLM이 호출할 수 있는 함수 또는 API 목록

## 기본 구성

`Agent` 생성자는 단일 구성 객체를 받습니다. 가장 일반적으로 사용하는 속성은 아래와 같습니다.

| 속성            | 필수 | 설명                                                                                               |
| --------------- | ---- | -------------------------------------------------------------------------------------------------- |
| `name`          | yes  | 사람이 읽을 수 있는 짧은 식별자                                                                    |
| `instructions`  | yes  | 시스템 프롬프트(문자열 **또는** 함수 – [동적 instructions](#dynamic-instructions) 참조)            |
| `model`         | no   | 모델 이름 **또는** 커스텀 [`Model`](/openai-agents-js/openai/agents/interfaces/model/) 구현        |
| `modelSettings` | no   | 튜닝 매개변수(temperature, top_p 등)                                                               |
| `tools`         | no   | 모델이 호출할 수 있는 [`Tool`](/openai-agents-js/openai/agents/type-aliases/tool/) 인스턴스의 배열 |

## 컨텍스트

에이전트는 **컨텍스트 타입에 대해 제네릭**입니다(즉, `Agent<TContext, TOutput>`). 컨텍스트는 당신이 생성하여 `Runner.run()`에 전달하는 의존성 주입 객체입니다. 이 객체는 모든 도구, 가드레일, 핸드오프 등에 전달되며 상태 저장 또는 공유 서비스(데이터베이스 연결, 사용자 메타데이터, 기능 플래그 등)를 제공하는 데 유용합니다.

## 출력 타입

기본적으로 에이전트는 **일반 텍스트**(`string`)를 반환합니다. 모델이 구조화된 객체를 반환하도록 하려면 `outputType` 속성을 지정하세요. SDK는 다음을 지원합니다:

1. [Zod](https://github.com/colinhacks/zod) 스키마(`z.object({...})`)

## 도구 사용 강제

도구를 제공한다고 해서 LLM이 반드시 호출하는 것은 아닙니다. `modelSettings.tool_choice`로 도구 사용을 **강제**할 수 있습니다:

1. `'auto'`(기본값) – LLM이 도구 사용 여부를 결정
2. `'required'` – LLM은 _반드시_ 도구를 호출해야 함(어떤 도구를 사용할지는 선택 가능)
3. `'none'` – LLM은 도구를 호출해서는 **안 됨**
4. 특정 도구 이름(예: `'calculator'`) – LLM은 해당 도구를 호출해야 함

### 무한 루프 방지

도구 호출 이후 SDK는 자동으로 `tool_choice`를 `'auto'`로 재설정합니다. 이는 모델이 도구를 반복적으로 호출하려는 무한 루프에 빠지는 것을 방지합니다. 이 동작은 `resetToolChoice` 플래그 또는
`toolUseBehavior` 구성을 통해 재정의할 수 있습니다:

- `'run_llm_again'`(기본값) – 도구 결과와 함께 LLM을 다시 실행
- `'stop_on_first_tool'` – 첫 번째 도구 결과를 최종 답변으로 처리
- `{ stopAtToolNames: ['my_tool'] }` – 나열된 도구 중 하나가 호출되면 종료
- `(context, toolResults) => ...` – 실행 종료 여부를 반환하는 커스텀 함수

```typescript
const agent = new Agent({
  ...,
  toolUseBehavior: 'stop_on_first_tool',
});
```
