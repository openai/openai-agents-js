Cloudflare Workers 및 기타 workerd 런타임에서는 전역 `WebSocket` 생성자를 사용해 아웃바운드 WebSocket을 열 수 없습니다. 이러한 환경에서 실시간 에이전트 연결을 간소화하기 위해, extensions 패키지는 내부적으로 `fetch()` 기반 업그레이드를 수행하는 전용 전송 방식을 제공합니다.

<Aside type="caution">
  이 어댑터는 아직 베타입니다. 엣지 케이스 이슈나 버그를 겪을 수 있습니다.
  문제를 발견하시면 [GitHub
  issues](https://github.com/openai/openai-agents-js/issues)로 보고해 주세요.
  신속히 수정하겠습니다. Workers에서 Node.js 스타일 API가 필요하다면
  `nodejs_compat` 활성화를 고려하세요.
</Aside>

## 설정

<Steps>

1. **extensions 패키지를 설치합니다.**

   ```bash
   npm install @openai/agents-extensions
   ```

2. **전송 방식을 생성하여 세션에 연결합니다.**

   <Code lang="typescript" code={cloudflareBasicExample} />

3. **`RealtimeSession`을(를) 연결합니다.**

   ```typescript
   await session.connect({ apiKey: 'your-openai-ephemeral-or-server-key' });
   ```

</Steps>

## 참고 사항

- Cloudflare 전송 방식은 내부적으로 `Upgrade: websocket`이 설정된 `fetch()`를 사용하며, 소켓의 `open` 이벤트를 대기하지 않아 workerd API와 일치합니다
- 이 전송 방식을 사용할 때에도 모든 `RealtimeSession` 기능(도구, 가드레일 등)은 동일하게 동작합니다
- 개발 중 상세 로그를 확인하려면 `DEBUG=openai-agents*`를 사용하세요
