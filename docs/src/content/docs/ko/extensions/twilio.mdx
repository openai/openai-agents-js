Twilio는 전화 통화의 원시 오디오를 WebSocket 서버로 보내는 [Media Streams API](https://www.twilio.com/docs/voice/media-streams)를 제공합니다. 이 설정을 사용하여 [음성 에이전트 개요](/openai-agents-js/ko/guides/voice-agents)를 Twilio에 연결할 수 있습니다. 기본 Realtime Session 전송 방식을 `websocket` 모드로 사용하여 Twilio에서 오는 이벤트를 Realtime Session에 연결할 수 있습니다. 다만, 웹 기반 대화보다 전화 통화는 자연스럽게 지연이 더 크므로 올바른 오디오 포맷을 설정하고 자체 인터럽션(중단 처리) 타이밍을 조정해야 합니다.

설정 경험을 개선하기 위해, Twilio 연결을 처리하고 인터럽션 및 오디오 포워딩까지 처리해 주는 전용 전송 레이어를 만들었습니다.

## 설정

<Steps>

1. **Twilio 계정과 Twilio 전화번호가 있어야 합니다.**

2. **Twilio로부터 이벤트를 받을 수 있는 WebSocket 서버를 설정합니다.**

   로컬에서 개발 중이라면, 다음과 같이 로컬 터널을 구성해야 합니다
   이는 [`ngrok`](https://ngrok.io/) 또는
   [Cloudflare Tunnel](https://developers.cloudflare.com/pages/how-to/preview-with-cloudflare-tunnel/)
   과 같은 로컬 터널을 구성하여 로컬 서버를 Twilio에서 접근 가능하게 만들어야 한다는 뜻입니다. `TwilioRealtimeTransportLayer`를 사용하여 Twilio에 연결할 수 있습니다.

3. **extensions 패키지를 설치하여 Twilio 어댑터를 설치합니다:**

   ```bash
   npm install @openai/agents-extensions
   ```

`RealtimeSession`에서 기대하는 모든 이벤트와 동작(함수 호출, 가드레일 등)이 정상 동작합니다. `RealtimeSession`을 음성 에이전트와 함께 사용하는 방법에 대해서는 [음성 에이전트 개요](/openai-agents-js/ko/guides/voice-agents)를 참고하세요.

## 팁 및 고려 사항

1. **속도가 가장 중요합니다.**

   Twilio로부터 필요한 모든 이벤트와 오디오를 받기 위해서는 WebSocket 연결에 대한 참조를 얻는 즉시 `TwilioRealtimeTransportLayer` 인스턴스를 생성하고, 바로 뒤이어 `session.connect()`를 호출해야 합니다.

2. **Twilio 원시 이벤트에 접근합니다.**

   Twilio가 보내는 원시 이벤트에 접근하려면 `RealtimeSession` 인스턴스에서 `transport_event` 이벤트를 수신하면 됩니다. Twilio에서 오는 모든 이벤트는 타입이 `twilio_message`이며, 원시 이벤트 데이터가 담긴 `message` 속성을 가집니다.

3. **디버그 로그를 확인합니다.**

   문제가 발생하여 내부 동작에 대한 추가 정보가 필요한 경우 `DEBUG=openai-agents*` 환경 변수를 사용하면 Agents SDK의 모든 디버그 로그가 표시됩니다. 또는 `DEBUG=openai-agents:extensions:twilio*`를 사용해 Twilio 어댑터에 대한 디버그 로그만 활성화할 수 있습니다.
