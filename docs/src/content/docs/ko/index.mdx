## 개요

[OpenAI Agents SDK for TypeScript](https://github.com/openai/openai-agents-js)는 소수의 추상화만으로 가볍고 사용하기 쉬운 패키지에서 에이전트형 AI 앱을 구축할 수 있게 해줍니다. 이는 이전 에이전트 실험인 [Swarm](https://github.com/openai/swarm/tree/main)을 프로덕션 수준으로 업그레이드한 것으로, [Python 버전](https://github.com/openai/openai-agents-python)도 제공합니다. Agents SDK는 매우 작은 기본 구성 요소 집합을 갖습니다:

- **Agents**: instructions 와 tools 를 갖춘 LLM
- **Agents as tools / Handoffs**: 특정 작업을 다른 에이전트에 위임할 수 있게 함
- **Guardrails**: 에이전트 입력을 검증할 수 있게 함

TypeScript 와 결합하면, 이들 기본 구성 요소만으로도 도구와 에이전트 간의 복잡한 관계를 표현하고, 가파른 학습 곡선 없이 실제 애플리케이션을 구축할 수 있습니다. 또한 SDK 는 에이전트 플로우를 시각화하고 디버그할 수 있는 내장 **tracing** 을 제공하며, 이를 평가하고 애플리케이션에 맞게 모델을 파인튜닝하는 것까지 지원합니다.

## Agents SDK를 사용하는 이유

SDK의 설계 원칙은 두 가지입니다:

1. 사용할 가치가 있을 만큼 충분한 기능을 제공하되, 학습이 빠르도록 기본 구성 요소는 최소화
2. 기본값으로도 훌륭하게 동작하되, 동작 방식을 정확히 커스터마이즈 가능

주요 기능은 다음과 같습니다:

- **에이전트 루프**: 도구 호출을 처리하고 결과를 LLM 에 다시 전달하며, 작업이 완료될 때까지 반복하는 내장 루프
- **TypeScript 우선**: 새로운 추상화를 배울 필요 없이 네이티브 TypeScript 언어 기능으로 에이전트를 오케스트레이션하고 체이닝
- **Agents as tools / Handoffs**: 여러 에이전트 간에 작업을 조정하고 위임하는 강력한 메커니즘
- **가드레일**: 에이전트 실행과 병렬로 입력 검증 및 안전성 점검을 수행하고, 실패 시 신속히 중단
- **함수 도구**: 어떤 TypeScript 함수든 자동 스키마 생성과 Zod 기반 검증으로 도구화
- **MCP 서버 도구 호출**: 함수 도구와 동일한 방식으로 동작하는 내장 MCP 서버 도구 통합
- **세션**: 에이전트 루프 내 작업 컨텍스트를 유지하는 지속형 메모리 계층
- **휴먼인더루프 (HITL)**: 에이전트 실행 전반에 사람을 개입시키는 내장 메커니즘
- **트레이싱**: 워크플로 시각화, 디버깅, 모니터링을 위한 내장 트레이싱과 OpenAI 평가, 파인튜닝, 증류 도구 연동 지원
- **Realtime Agents**: 자동 인터럽션(중단 처리) 감지, 컨텍스트 관리, 가드레일 등 기능을 갖춘 강력한 음성 에이전트 구축

## 설치

```bash
npm install @openai/agents zod@3
```
